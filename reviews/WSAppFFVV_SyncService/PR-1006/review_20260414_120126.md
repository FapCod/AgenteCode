## 📋 Resumen del PR

**¿Que hace este PR?**
Completa la migracion de credenciales y connection strings desde archivos de configuracion local (Web.config) hacia AWS Secrets Manager. Implementa un patron Strategy con orchestrators especificos para diferentes tipos de secretos (connection strings, app settings, registro de paises).

**Archivos principales modificados:**
- [NEW] `SecretFactory/AwsSecretsManagerStrategy.cs` - Cliente AWS Secrets Manager con cache thread-safe
- [NEW] `SecretFactory/ConnectionStrategyOrchestrator.cs` - Orquestador para connection strings
- [NEW] `SecretFactory/AppSettingsOrchestrator.cs` - Orquestador para app settings
- [NEW] `SecretFactory/DbPaisConfiguration.cs` - Parser de configuracion de BD por pais
- [NEW] `SecretFactory/RegistroDbPaisesOrchestrator.cs` - Orquestador para BD paises SQL
- [NEW] `SecretFactory/RegistroMongoPaisesOrchestrator.cs` - Orquestador para BD paises Mongo
- [NEW] `SecretFactory/RegistroMongoFFVVPaisesOrchestrator.cs` - Orquestador para BD FFVV Mongo
- [MODIFY] `BelcorpConnectionStringProvider.cs` - Adaptado para usar orchestrators
- [MODIFY] `DataPackage.cs` - DI actualizada para usar secretos de AWS
- [MODIFY] `AmazonS3Util.cs` - Inicializacion de keys movida al constructor
- [MODIFY] `Web.config` - Removidas credenciales, agregada config de AWS
- [DELETE] `app/packages/AWSSDK.*` - Paquetes legacy removidos (389,958 lineas)

**¿Que podria impactar?**
- Todos los servicios ahora requieren acceso a AWS Secrets Manager para iniciar
- El startup de la aplicacion depende de la disponibilidad de AWS Secrets Manager
- Si fallan las credenciales AWS o el secreto no existe, la aplicacion no iniciara

**Impacto:**
- SEGURIDAD: Alto impacto positivo - credenciales ya no estan en codigo fuente
- ARQUITECTURA: Cambio significativo en como se obtienen configuraciones
- DEPENDENCIAS: Actualizacion de AWS SDK a version 3.7.500
