## 🟡 Problemas Importantes (P1 - High Priority)

1. **Thread Safety Issue - Lock Estatico con Cache de Instancia**
   - **Archivo**: `AwsSecretsManagerStrategy.cs` lineas 17-19
   - **Problema**: El campo `_connectionStringsCache` es de **instancia**, pero el lock `CacheLock` es **estatico**. Esto crea una inconsistencia donde multiples instancias de la clase comparten el mismo lock pero tienen caches diferentes.
   - **Impacto**: Condiciones de carrera potenciales. Una instancia podria verificar `if (_connectionStringsCache != null)` mientras otra esta modificando su propio cache, sin proteccion real. Ademas, multiples instancias podrian intentar cargar el mismo secreto de AWS simultaneamente si se crean antes de que cualquiera complete la carga.
   - **Impacto en score**: 6 puntos (Robustez)
   - **Tiempo est.**: 15 minutos
   - **Categoria afectada**: Robustez
   - **Codigo actual**:

     ```csharp
     private Dictionary<string, string> _connectionStringsCache;
     private static readonly object CacheLock = new object();
     ```

   - **Solucion sugerida**: Hacer el cache tambien estatico (si se desea compartir entre instancias) o hacer el lock de instancia (si cada instancia debe tener su propio cache aislado). Dado que los secretos de AWS son unicos por `_secretName`, se recomienda usar un `ConcurrentDictionary` estatico compartido:

     ```csharp
     private static readonly ConcurrentDictionary<string, Dictionary<string, string>> _globalSecretsCache 
         = new ConcurrentDictionary<string, Dictionary<string, string>>();
     ```

2. **Import No Utilizado**
   - **Archivo**: `AwsSecretsManagerStrategy.cs` linea 8
   - **Problema**: El namespace `System.Threading.Tasks` esta importado pero no se utiliza en ninguna parte del codigo.
   - **Impacto**: Codigo muerto que afecta la mantenibilidad y puede confundir sobre si hay logica async pendiente.
   - **Impacto en score**: 2 puntos (Mantenibilidad)
   - **Tiempo est.**: 1 minuto
   - **Categoria afectada**: Mantenibilidad
   - **Codigo actual**:

     ```csharp
     using System.Threading.Tasks;
     ```

   - **Solucion sugerida**: Eliminar el import no utilizado.

3. **Inicializacion de Campos con Llamadas Externas Potencialmente Fallidas**
   - **Archivo**: `AmazonS3Util.cs` lineas 17-18
   - **Problema**: Los campos `S3_AccessKey` y `S3_SecretKey` se inicializan directamente con llamadas a `AppSettingsOrchestrator.GetAppSetting()`. Si AWS Secrets Manager falla durante la construccion del objeto, la excepcion se propagara desde el inicializador de campo.
   - **Impacto**: Las excepciones lanzadas desde inicializadores de campo son mas dificiles de manejar y debuggear. Ademas, si falla, todas las instancias fallaran sin posibilidad de retry o fallback.
   - **Impacto en score**: 4 puntos (Robustez)
   - **Tiempo est.**: 10 minutos
   - **Categoria afectada**: Robustez
   - **Codigo actual**:

     ```csharp
     private string S3_AccessKey = AppSettingsOrchestrator.GetAppSetting("S3_AccessKey");
     private string S3_SecretKey = AppSettingsOrchestrator.GetAppSetting("S3_SecretKey");
     ```

   - **Solucion sugerida**: Mover la inicializacion al constructor o usar lazy initialization.

     ```csharp
     private string _s3AccessKey;
     private string _s3SecretKey;
     
     public AmazonS3Util()
     {
         _s3AccessKey = AppSettingsOrchestrator.GetAppSetting("S3_AccessKey");
         _s3SecretKey = AppSettingsOrchestrator.GetAppSetting("S3_SecretKey");
     }
     ```

## 📋 Resumen del PR

**¿Que hace este PR?**
Migra la gestion de connection strings y configuraciones sensibles desde archivos de configuracion local (Web.config) hacia AWS Secrets Manager, implementando un patron Strategy con orquestadores especificos para diferentes tipos de secretos.

**Archivos principales modificados:**
- [NEW] `SecretFactory/AwsSecretsManagerStrategy.cs` - Cliente AWS Secrets Manager con cache
- [NEW] `SecretFactory/ConnectionStrategyOrchestrator.cs` - Orquestador para connection strings
- [NEW] `SecretFactory/AppSettingsOrchestrator.cs` - Orquestador para app settings
- [NEW] `SecretFactory/DbPaisConfiguration.cs` - Parser de configuracion de BD por pais
- [MODIFY] `BelcorpConnectionStringProvider.cs` - Adaptado para usar orchestrators
- [MODIFY] `DataPackage.cs` - DI actualizada para usar secretos de AWS
- [MODIFY] `Web.config` - Removidas credenciales, agregada config de AWS
- [DELETE] `app/packages/AWSSDK.*` - Paquetes legacy removidos

**¿Que podria impactar?**
- Todos los servicios que dependan de connection strings ahora requieren acceso a AWS Secrets Manager
- El startup de la aplicacion ahora depende de la disponibilidad de AWS Secrets Manager
- Si fallan las credenciales AWS o el secreto no existe, la aplicacion no iniciara

**Impacto:**
- SEGURIDAD: Alto impacto positivo - credenciales ya no estan en codigo fuente
- ARQUITECTURA: Cambio significativo en como se obtienen configuraciones
- DEPENDENCIAS: Actualizacion de AWS SDK a version 3.7.500
