## 🔴 Problemas Criticos (P0 - Blocking)

1. **Inconsistencia de Key en AppSettings - APLICACION NO INICIARA**
   - **Archivo**: `app/Belcorp.FFVV.Aplicacion/Web.config` linea 69 + `app/Belcorp.FFVV.Infr.Datos/SecretFactory/AwsSecretsManagerStrategy.cs` linea 12
   - **Problema**: El codigo C# busca `AWS_SECRET_ENVIRONMENT` (con 'N') pero el archivo `Web.config` tiene `AWS_SECRET_ENVIROMENT` (sin 'N' - typo). Esta inconsistencia causara que `ConfigurationManager.AppSettings["AWS_SECRET_ENVIRONMENT"]` retorne `null`, lo que dispara un `InvalidOperationException` y **la aplicacion no podra iniciar**.
   - **Riesgo**: La aplicacion fallara inmediatamente al desplegarse. Ningun servicio WCF funcionara.
   - **Impacto en score**: -15 puntos (Funcionalidad + Robustez)
   - **Tiempo est.**: 5 minutos
   - **Categoria afectada**: Funcionalidad, Robustez
   - **Codigo actual en Web.config**:

     ```xml
     <add key="AWS_SECRET_ENVIROMENT" value="qa" />
     ```

   - **Codigo actual en AwsSecretsManagerStrategy.cs**:

     ```csharp
     string secretEnvironment = ConfigurationManager.AppSettings["AWS_SECRET_ENVIRONMENT"];
     if (string.IsNullOrEmpty(secretEnvironment))
     {
         throw new InvalidOperationException("AWS_SECRET_ENVIRONMENT no esta configurado en AppSettings.");
     }
     ```

   - **Solucion sugerida**: Corregir el typo en `Web.config` (y cualquier otro archivo .config) para que coincida con el codigo:

     ```xml
     <add key="AWS_SECRET_ENVIRONMENT" value="qa" />
     ```

## 🟡 Problemas Importantes (P1 - High Priority)

1. **Sin tolerancia a fallos si AWS Secrets Manager no esta disponible**
   - **Archivo**: `app/Belcorp.FFVV.Infr.Datos/SecretFactory/AwsSecretsManagerStrategy.cs` lineas 63-67
   - **Problema**: Si AWS Secrets Manager no esta disponible (error de red, credenciales invalidas, etc.), la aplicacion lanza excepcion y **no puede iniciar**. No hay fallback a configuracion local.
   - **Impacto**: Un problema temporal de conectividad con AWS impide completamente el inicio de la aplicacion. Deberia degradar elegantemente.
   - **Impacto en score**: -5 puntos (Robustez)
   - **Tiempo est.**: 30 minutos
   - **Categoria afectada**: Robustez
   - **Codigo actual**:

     ```csharp
     catch (Exception ex)
     {
         System.Diagnostics.Trace.TraceError($"Error fatal al cargar secretos desde AWS ({_secretName}): {ex.Message}");
         throw; // Detiene la aplicacion completamente
     }
     ```

   - **Solucion sugerida**: Implementar fallback a `ConfigurationManager.ConnectionStrings` cuando AWS falle.

     ```csharp
     catch (Exception ex)
     {
         System.Diagnostics.Trace.TraceWarning($"No se pudo conectar a AWS Secrets Manager ({_secretName}): {ex.Message}. Usando fallback local.");
         _connectionStringsCache = new Dictionary<string, string>();
         // El ConnectionFactory ya tiene logica de fallback a ConfigurationManager
     }
     ```

## 📋 Resumen del PR

**¿Que hace este PR?**
Migra la gestion de credenciales y connection strings desde archivos `.config` hardcodeados hacia AWS Secrets Manager. Implementa el patron Strategy con clases orquestadoras que obtienen los secretos de AWS en tiempo de ejecucion. Este es un re-review del PR original.

**Archivos principales modificados:**
- [NEW] `SecretFactory/AwsSecretsManagerStrategy.cs` - Implementacion del cliente AWS Secrets Manager con cache
- [NEW] `SecretFactory/ConnectionStrategyOrchestrator.cs` - Orquestador para connection strings
- [NEW] `SecretFactory/AppSettingsOrchestrator.cs` - Orquestador para app settings
- [NEW] `SecretFactory/IConnectionStringStrategy.cs` - Interfaz del patron Strategy

- [MODIFY] `ValidacionCrediticiaService.svc.cs` - Migracion de credenciales Equifax
- [MODIFY] `Web.config` - Eliminacion de credenciales hardcodeadas, adicion de keys AWS (CON TYPO PENDIENTE)

- La aplicacion ahora **depende de AWS Secrets Manager** para iniciar. Si AWS no esta disponible, la aplicacion no iniciara.
- Los secretos deben existir en AWS Secrets Manager con las keys correctas antes del despliegue.

**Impacto:**
- 🔒 **SEGURIDAD**: Mejora significativa - elimina credenciales del codigo fuente
- ⚡ **PERFORMANCE**: Neutral - cache de secretos en memoria
