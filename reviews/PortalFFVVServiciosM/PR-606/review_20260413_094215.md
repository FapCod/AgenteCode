## 🟡 Problemas Importantes (P1 - High Priority)

1. **Uso de `.GetAwaiter().GetResult()` - Riesgo de Deadlock**
   - **Archivo**: `app/Belcorp.FFVV.Infr.Datos/SecretFactory/AwsSecretsManagerStrategy.cs` línea 58
   - **Problema**: Se usa `.GetAwaiter().GetResult()` para llamar `GetSecretValueAsync` de forma síncrona. Esto puede causar deadlocks en contextos ASP.NET donde hay SynchronizationContext.
   - **Impacto**: En escenarios de carga, la aplicación podría bloquearse indefinidamente al iniciar.
   - **Impacto en score**: -6 puntos (Robustez)
   - **Tiempo est.**: 15 minutos
   - **Categoria afectada**: Robustez
   - **Código actual**:

     ```csharp
     var response = _client.GetSecretValueAsync(request).GetAwaiter().GetResult();
     ```

   - **Solución sugerida**: Usar el método síncrono `GetSecretValue()` o implementar un patrón async-friendly durante la inicialización.

     ```csharp
     var response = _client.GetSecretValue(request);
     ```

2. **Typo en nombre de AppSetting: "ENVIROMENT" → "ENVIRONMENT"**
   - **Archivo**: `app/Belcorp.FFVV.Infr.Datos/SecretFactory/AwsSecretsManagerStrategy.cs` línea 18
   - **Problema**: La key `AWS_SECRET_ENVIROMENT` tiene un typo (falta la 'N'). Esto afecta la consistencia y puede causar confusión.
   - **Impacto**: Inconsistencia con estándares de naming. Futuras integraciones podrían usar el nombre correcto y fallar.
   - **Impacto en score**: -3 puntos (Mantenibilidad)
   - **Tiempo est.**: 5 minutos
   - **Categoria afectada**: Mantenibilidad
   - **Código actual**:

     ```csharp
     string secretEnviroment = ConfigurationManager.AppSettings["AWS_SECRET_ENVIROMENT"];
     ```

   - **Solución sugerida**: Corregir el typo en todos los archivos de configuración y código.

     ```csharp
     string secretEnvironment = ConfigurationManager.AppSettings["AWS_SECRET_ENVIRONMENT"];
     ```
     
     > También actualizar `Web.config` y otros `.config` donde aparece `AWS_SECRET_ENVIROMENT`.

3. **Sin tolerancia a fallos si AWS Secrets Manager no está disponible**
   - **Archivo**: `app/Belcorp.FFVV.Infr.Datos/SecretFactory/AwsSecretsManagerStrategy.cs` línea 51-62
   - **Problema**: Si AWS Secrets Manager no está disponible (error de red, credenciales inválidas, etc.), la aplicación lanza excepción y **no puede iniciar**. No hay fallback a configuración local.
   - **Impacto**: Un problema temporal de conectividad con AWS impide completamente el inicio de la aplicación. Debería degradar elegantemente.
   - **Impacto en score**: -5 puntos (Robustez)
   - **Tiempo est.**: 30 minutos
   - **Categoria afectada**: Robustez
   - **Código actual**:

     ```csharp
     catch (Exception ex)
     {
         System.Diagnostics.Trace.TraceError($"Error fatal al cargar secretos desde AWS ({_secretName}): {ex.Message}");
         throw; // ← Detiene la aplicación completamente
     }
     ```

   - **Solución sugerida**: Implementar fallback a `ConfigurationManager.ConnectionStrings` cuando AWS falle, permitiendo que la aplicación inicie con configuración local como último recurso.

     ```csharp
     catch (Exception ex)
     {
         System.Diagnostics.Trace.TraceWarning($"No se pudo conectar a AWS Secrets Manager ({_secretName}): {ex.Message}. Usando fallback local.");
         _connectionStringsCache = new Dictionary<string, string>(); // Fallback vacío, se usará config local
     }
     ```

4. **Validación de secretName ocurre DESPUÉS del string.Format**
   - **Archivo**: `app/Belcorp.FFVV.Infr.Datos/SecretFactory/AwsSecretsManagerStrategy.cs` líneas 17-22
   - **Problema**: Se valida `string.IsNullOrEmpty(secretName)` después de hacer `string.Format(secretName, secretEnviroment)`. Si el secretName original era vacío, el Format ya habría fallado o producido resultado inesperado.
   - **Impacto**: La validación no protege correctamente. Además si `secretEnviroment` es null, el resultado será un nombre de secreto con `{0}` sin reemplazar.
   - **Impacto en score**: -3 puntos (Robustez)
   - **Tiempo est.**: 10 minutos
   - **Categoria afectada**: Robustez
   - **Código actual**:

     ```csharp
     string secretEnviroment = ConfigurationManager.AppSettings["AWS_SECRET_ENVIROMENT"];
     secretName = string.Format(secretName, secretEnviroment);
     if (string.IsNullOrEmpty(secretName)) // ← Validación tardía
     {
         throw new ArgumentNullException(...);
     }
     ```

   - **Solución sugerida**: Validar ANTES del Format y también validar que el environment no sea vacío.

     ```csharp
     if (string.IsNullOrEmpty(secretName))
     {
         throw new ArgumentNullException(nameof(secretName), "El nombre del secreto no puede ser nulo o vacío.");
     }
     
     string secretEnvironment = ConfigurationManager.AppSettings["AWS_SECRET_ENVIRONMENT"];
     if (string.IsNullOrEmpty(secretEnvironment))
     {
         throw new InvalidOperationException("AWS_SECRET_ENVIRONMENT no está configurado en AppSettings.");
     }
     
     _secretName = string.Format(secretName, secretEnvironment);
     ```

## 📋 Resumen del PR

**¿Qué hace este PR?**
Migra la gestión de credenciales y connection strings desde archivos `.config` hardcodeados hacia AWS Secrets Manager. Implementa el patrón Strategy con clases orquestadoras (`ConnectionStrategyOrchestrator`, `AppSettingsOrchestrator`) que obtienen los secretos de AWS en tiempo de ejecución.

**Archivos principales modificados:**
- [NEW] `SecretFactory/AwsSecretsManagerStrategy.cs` - Implementación del cliente AWS Secrets Manager con caché
- [NEW] `SecretFactory/ConnectionStrategyOrchestrator.cs` - Orquestador para connection strings
- [NEW] `SecretFactory/AppSettingsOrchestrator.cs` - Orquestador para app settings
- [NEW] `SecretFactory/IConnectionStringStrategy.cs` - Interfaz del patrón Strategy
- [MODIFY] `FFVVFlexiPago.svc.cs` - Migración de 17 usos de ConfigurationManager
- [MODIFY] `ValidacionCrediticiaService.svc.cs` - Migración de credenciales Equifax
- [MODIFY] `Web.config` - Eliminación de credenciales hardcodeadas, adición de keys AWS

**¿Qué podría impactar?**
- La aplicación ahora **depende de AWS Secrets Manager** para iniciar. Si AWS no está disponible, la aplicación no iniciará.
- Se requiere configurar las variables `AWS_ACCESS_KEY_ID_CUSTOM`, `AWS_SECRET_ACCESS_KEY_CUSTOM`, `AWS_REGION` y `AWS_SECRET_ENVIROMENT` en producción.
- Los secretos deben existir en AWS Secrets Manager con las keys correctas antes del despliegue.

**Impacto:**
- 🔒 **SEGURIDAD**: Mejora significativa - elimina credenciales del código fuente
- ⚡ **PERFORMANCE**: Neutral - caché de secretos en memoria
- 🏗️ **ARQUITECTURA**: Positivo - patrón extensible para futuras integraciones
