## 🟡 Problemas Importantes (P1 - High Priority)

1. **[Falta manejo de errores en JSON.parse]**
   - **Archivo**: `src/utils/env.ts` línea 4-5
   - **Problema**: Si `process.env.APP_CONFIG` contiene JSON inválido, `JSON.parse()` lanzará una excepción no manejada que crasheará la aplicación durante el startup.
   - **Impacto**: Crash de la aplicación al iniciar si la variable de entorno está malformada.
   - **Impacto en score**: -4 puntos
   - **Tiempo est.**: 5 minutos
   - **Categoria afectada**: Robustez
   - **Código actual**:

     ```typescript
     function getEnv(): any {
       if (process.env.APP_CONFIG) {
         return JSON.parse(process.env.APP_CONFIG)
       }
     ```

   - **Solución sugerida**: Envolver en try-catch con mensaje de error descriptivo

     ```typescript
     function getEnv(): any {
       if (process.env.APP_CONFIG) {
         try {
           return JSON.parse(process.env.APP_CONFIG)
         } catch (error) {
           throw new Error(`Invalid JSON in APP_CONFIG environment variable: ${error.message}`)
         }
       }
       return yenv()
     }
     ```

2. **[JSON parsing repetido sin caché]**
   - **Archivo**: `src/utils/env.ts` líneas 3-8
   - **Problema**: La función `getEnv()` se invoca en múltiples archivos (`db.ts`, `dbSql.ts`, `server.ts`, controllers, etc.) al inicio de cada módulo. Cada llamada ejecuta `JSON.parse()` sobre el mismo string, lo cual es trabajo redundante.
   - **Impacto**: Impacto mínimo en performance de startup, pero es un patrón ineficiente que puede escalar mal en proyectos más grandes.
   - **Impacto en score**: -4 puntos
   - **Tiempo est.**: 5 minutos
   - **Categoria afectada**: Escalabilidad
   - **Código actual**:

     ```typescript
     function getEnv(): any {
       if (process.env.APP_CONFIG) {
         return JSON.parse(process.env.APP_CONFIG)
       }
       const env=yenv();
       return env;
     }
     ```

   - **Solución sugerida**: Implementar patrón singleton/caché

     ```typescript
     let cachedConfig: any = null

     function getEnv(): any {
       if (cachedConfig) {
         return cachedConfig
       }

       if (process.env.APP_CONFIG) {
         try {
           cachedConfig = JSON.parse(process.env.APP_CONFIG)
         } catch (error) {
           throw new Error(`Invalid JSON in APP_CONFIG: ${error.message}`)
         }
       } else {
         cachedConfig = yenv()
       }

       return cachedConfig
     }

     export default getEnv
     ```
