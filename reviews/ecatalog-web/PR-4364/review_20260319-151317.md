## 🔴 Problemas Críticos (P0 - Blocking)

> *Issues de seguridad, crashes, o violaciones de reglas estrictas. Deben corregirse obligatoriamente.*

1. **Bug Lógico: `cart.totalItems` se incrementa antes del filtro de exclusión**
   - **Archivo**: `src/graphql/server/dataSources/CartAPI.js` línea 287-305 (aprox.)
   - **Problema**: El acumulador `cart.totalItems += totalQuantity` se ejecuta **antes** del guard `if (!shouldIncludeProduct(...)) { continue; }`. Esto significa que cuando la feature está habilitada y un producto de tipo set-hijo es excluido del cálculo del precio (`cart.total`), su cantidad **sí sigue sumándose** al contador de ítems del carrito. El carrito mostrará más ítems de los que realmente se están cobrando, causando una inconsistencia visible para el usuario.
   - **Riesgo**: Bug lógico silencioso — el total monetario será correcto, pero el número de ítems mostrado (`cart.totalItems`) estará inflado. Afecta la experiencia del usuario y puede generar confusión en auditoría de pedidos.
   - **Impacto en score**: -8 puntos (Funcionalidad)
   - **Tiempo est.**: 5 minutos
   - **Categoria afectada**: Funcionalidad
   - **Código actual**:

     ```javascript
     totalQuantity = products[i].quantity < 100 ? products[i].quantity : 99;
     cart.totalItems += totalQuantity;  // ← Se suma ANTES del filtro
     if (!shouldIncludeProduct(products[i].product, products[i])) {
       continue;  // Solo afecta cart.total, no cart.totalItems
     }
     const itemOfferPrice = ...
     cart.total += toNumber(itemOfferPrice) * totalQuantity;
     ```

   - **Solución sugerida**: Mover `cart.totalItems += totalQuantity` después de la validación de `shouldIncludeProduct`, o invocar `shouldIncludeProduct` antes de acumular el conteo de ítems.

     ```javascript
     totalQuantity = products[i].quantity < 100 ? products[i].quantity : 99;
     if (!shouldIncludeProduct(products[i].product, products[i])) {
       continue;  // Excluye tanto del total de precio como del conteo de ítems
     }
     cart.totalItems += totalQuantity;  // ← Solo suma si el producto es válido
     const itemOfferPrice =
       products[i]?.product?.pricing?.setOfferPrice ??
       products[i]?.product?.pricing?.offerPrice;
     cart.total += toNumber(itemOfferPrice) * totalQuantity;
     ```

## 📋 Resumen del PR
**¿Qué hace este PR?**
Introduce un feature flag (`feature_calculo_total_cart`) que controla si los productos hijos de estrategias de tipo "set" (codes 3 y 7) deben ser excluidos del cálculo del total del carrito. Cuando el flag está activo, los productos que son parte de un set-hijo se omiten del precio final, evitando doble conteo de sets.

**Archivos principales modificados:**
- [MODIFY] `src/graphql/server/dataSources/CartAPI.js` - Agrega lógica de feature flag y filtrado de productos set en el cálculo del carrito

**¿Qué podría impactar?**
- El cálculo del total monetario del carrito (`cart.total`) y el conteo de ítems (`cart.totalItems`) para usuarios con productos de tipo set.
- Cualquier componente que consuma estos campos del carrito (UI de resumen de compra, validaciones de precio, etc.).

**Impacto:**
- FUNCIONALIDAD (bug lógico en conteo de ítems)
- PERFORMANCE (mínimo impacto — llamada adicional al servicio de feature flags por cada request de totales de carrito)
