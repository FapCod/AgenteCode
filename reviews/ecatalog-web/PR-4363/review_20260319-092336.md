## 🟢 Recomendaciones (P2/P3 - Medium/Low)

**P2 - Medium Priority:**

1. 💡 **Variable `error` en `catch` causa shadowing con el `error` del scope externo**
   - **Archivo**: `src/graphql/server/dataSources/ConsultantAPI.js` líneas ~51, ~82, ~108
   - **Impacto en score**: -3 puntos (Mantenibilidad)
   - **Tiempo est.**: 5 minutos
   - **Categoria afectada**: Mantenibilidad
   - **Código actual**:

     ```js
     } catch (error) {
       console.error('Error al enviar log a Elastic:', error);
     }
     ```

   - **Solución sugerida**: Usar un nombre descriptivo que evite el shadowing con la variable `error` del `catch` externo o del scope del método:

     ```js
     } catch (logError) {
       console.error('Error al enviar log a Elastic:', logError);
     }
     ```

**P3 - Low Priority:**

1. ℹ️ **HostName hardcodeado como literal `'server'`** — Considerar hacerlo dinámico
   - **Archivo**: `src/lib/utils/errorLogger.js` línea ~118
   - **Impacto en score**: 0 puntos (informativo)
   - **Tiempo est.**: 5 minutos
   - **Categoria afectada**: Mantenibilidad
   - **Código actual**:

     ```js
     HostName: 'server',
     ```

   - **Solución sugerida**: Utilizar variable de entorno para mayor trazabilidad por instancia de servidor:

     ```js
     HostName: process.env.HOSTNAME || 'server',
     ```

## 📋 Resumen del PR
**¿Qué hace este PR?**
Agrega validación del formato de slug de consultora vía feature flag (`feature_validate_slug_format`) y registro de logs contextuales a Kibana vía feature flag (`feature_register_log_kibana`). Las llamadas a feature flags se ejecutan en paralelo con `Promise.all`. Todas las operaciones de logging son best-effort (protegidas con try-catch).

**Archivos principales modificados:**
- [MODIFY] `src/graphql/server/dataSources/ConsultantAPI.js` - Feature flags paralelos, validación de slug con RegEx, logging best-effort con try-catch
- [MODIFY] `src/lib/utils/errorLogger.js` - Nuevo método `buildLogDataWithoutError`

**¿Qué podría impactar?**
El método `getConsultantByCodeOrDocument` es crítico para la apertura del catálogo digital de consultoras. Los cambios añaden resiliencia: incluso si Elastic cae, el flujo de negocio continúa correctamente.

**Impacto:**
- PERFORMANCE: Mejora de latencia por paralelización de feature flags
- ROBUSTEZ: Logging best-effort protege el flujo crítico ante fallos de Elastic
- OBSERVABILIDAD: Trazabilidad mejorada de slugs inválidos y skips de MongoDB en Kibana
