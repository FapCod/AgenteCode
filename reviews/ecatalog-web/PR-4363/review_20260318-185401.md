## 🟡 Problemas Importantes (P1 - High Priority)
> *Problemas de performance, bugs probables o deuda técnica alta.*

1. **`apiError.statusCode` es `undefined` en el log (Bug Lógico)**
   - **Archivo**: `src/graphql/server/dataSources/ConsultantAPI.js` líneas ~44-54
   - **Problema**: El log de `saveLog` se invoca con `status: apiError.statusCode`, pero `apiError.statusCode` se asigna en la línea SIGUIENTE al bloque del log (`apiError.statusCode = error.extensions?.response?.status || 500`). El campo `status` en el log siempre será `undefined`.
   - **Impacto**: Los logs enviados a Kibana tendrán el campo `status` vacío, dificultando el diagnóstico de errores.
   - **Impacto en score**: -8 puntos (Funcionalidad, Robustez)
   - **Tiempo est.**: 10 minutos
   - **Categoria afectada**: Funcionalidad / Robustez
   - **Código actual**:

     ```js
     const apiError = new Error(`Error fetching consultant from API: ${error.message || 'Unknown error'}`);

     if (shouldRegisterLogKibana) {
       await errorLogger.saveLog(apiError, {
         countryCode,
         params,
         error: apiError.message,
         status: apiError.statusCode,    // ← UNDEFINED: aún no se asignó
         body: apiError.originalError    // ← UNDEFINED: aún no se asignó
       });
     }

     apiError.statusCode = error.extensions?.response?.status || 500; // ← se asigna DESPUÉS
     apiError.originalError = error;
     throw apiError;
     ```

   - **Solución sugerida**: Asignar las propiedades antes del bloque de log.

     ```js
     const apiError = new Error(`Error fetching consultant from API: ${error.message || 'Unknown error'}`);
     apiError.statusCode = error.extensions?.response?.status || 500;
     apiError.originalError = error;

     if (shouldRegisterLogKibana) {
       await errorLogger.saveLog(apiError, {
         countryCode,
         params,
         error: apiError.message,
         status: apiError.statusCode,    // ✅ ya tiene valor
         body: apiError.originalError    // ✅ ya tiene valor
       });
     }

     throw apiError;
     ```

2. **`sendToElastic` sin tolerancia a fallos (Dependencia Dura)**
   - **Archivo**: `src/graphql/server/dataSources/ConsultantAPI.js` líneas ~78 y ~100
   - **Problema**: Las llamadas a `errorLogger.sendToElastic(context)` y `errorLogger.saveLog(apiError, ...)` no tienen manejo de errores. Si Elasticsearch no está disponible, la excepción se propagará hacia el flujo principal.
   - **Impacto**: Un timeout o error de red con Kibana/Elastic puede romper el flujo de validación de slug o el flujo del consultor, causando una experiencia degradada al usuario final.
   - **Impacto en score**: -10 puntos (Robustez, Escalabilidad)
   - **Tiempo est.**: 15 minutos
   - **Categoria afectada**: Robustez / Escalabilidad
   - **Código actual**:

     ```js
     await errorLogger.sendToElastic(context); // sin try-catch
     ```

   - **Solución sugerida**: Envolver las llamadas al logger en `try-catch` para que fallen silenciosamente (best-effort logging):

     ```js
     try {
       await errorLogger.sendToElastic(context);
     } catch (logError) {
       console.error('Error al enviar log a Elastic:', logError);
     }
     ```

     Aplicar igualmente para `errorLogger.saveLog(...)`.

3. **Llamadas secuenciales a Feature Flags (Waterfall Anti-pattern)**
   - **Archivo**: `src/graphql/server/dataSources/ConsultantAPI.js` líneas ~25-31
   - **Problema**: Dos llamadas `await getFeatureFlag()` en serie cuando son completamente independientes entre sí. Se suman las latencias de ambas llamadas.
   - **Impacto**: Incremento de latencia en cada invocación a `getConsultantByCodeOrDocument`. Si cada llamada tarda 20ms, se añaden 40ms en serie cuando podrían ejecutarse en paralelo.
   - **Impacto en score**: -5 puntos (Escalabilidad)
   - **Tiempo est.**: 10 minutos
   - **Categoria afectada**: Escalabilidad
   - **Código actual**:

     ```js
     const featureFlagValidateSlugFormat = await featureFlagAPIV2.getFeatureFlag(
       'feature_validate_slug_format'
     );

     const featureFlagRegisterLogKibana = await featureFlagAPIV2.getFeatureFlag(
       'feature_register_log_kibana'
     );
     ```

   - **Solución sugerida**: Usar `Promise.all` para ejecutarlas en paralelo:

     ```js
     const [featureFlagValidateSlugFormat, featureFlagRegisterLogKibana] = await Promise.all([
       featureFlagAPIV2.getFeatureFlag('feature_validate_slug_format'),
       featureFlagAPIV2.getFeatureFlag('feature_register_log_kibana')
     ]);
     ```

## 🎓 Aprendizajes Clave
- ❌ **Evitar**: Leer propiedades de un objeto antes de asignarlas — `apiError.statusCode` era `undefined` en el momento del log.
- ✅ **Preferir**: Asignar todas las propiedades de un objeto antes de usarlo en cualquier operación downstream.
- ❌ **Evitar**: Llamadas `await` secuenciales a operaciones independientes — suman latencias innecesariamente.
- ✅ **Preferir**: `Promise.all([...])` para operaciones independientes en paralelo.
- ❌ **Evitar**: Dependencias externas "duras" en logging — si Kibana cae, no debe romper el flujo de negocio.
- ✅ **Preferir**: Try-catch alrededor de operaciones de logging para hacer el logger "best-effort".

## 📋 Resumen del PR
**¿Qué hace este PR?**
Agrega validación del formato de slug de consultora mediante feature flag (`feature_validate_slug_format`) y registro de logs contextuales a Kibana/Elasticsearch mediante otro feature flag (`feature_register_log_kibana`). Además incluye una limpieza de formato en `errorLogger.js`.

**Archivos principales modificados:**
- [MODIFY] `src/graphql/server/dataSources/ConsultantAPI.js` - Agrega lógica de feature flags, validación de slug con RegEx y envío de logs a Kibana
- [MODIFY] `src/lib/utils/errorLogger.js` - Nuevo método `buildLogDataWithoutError`, clean-up de formato general

**¿Qué podría impactar?**
El método `getConsultantByCodeOrDocument` es crítico en el flujo de acceso al catálogo digital de consultoras. Cualquier excepción no controlada en las llamadas al logger podría bloquear la apertura del catálogo.

**Impacto:**
- PERFORMANCE: Latencia adicional por llamadas secuenciales a feature flags
- ROBUSTEZ: Dependencia no controlada de Elasticsearch en flujo crítico
- LOGGING: Datos de log incompletos (`statusCode` undefined)
