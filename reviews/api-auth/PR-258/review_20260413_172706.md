## 🔴 Problemas Críticos (P0 - Blocking)

1. **[Bug de Flujo en tokenExchange - Respuesta Sobrescrita]**
   - **Archivo**: `api-auth/src/controllers/Auth.js` líneas 145-165
   - **Problema**: Después del bloque `catch`, el código continúa ejecutando `ctx.status = 200` y `ctx.body = internalToken`, sobrescribiendo cualquier respuesta de error configurada en el catch. Si `exchangeForInternalToken()` falla, el cliente recibirá un status 200 con `body: undefined`.
   - **Riesgo**: Errores de OAuth se reportarán como éxito (200 OK), causando comportamiento inesperado en el frontend y dificultando el debugging.
   - **Impacto en score**: -6 puntos
   - **Tiempo est.**: 5 minutos
   - **Categoría afectada**: Funcionalidad/Robustez
   - **Código actual**:

     ```javascript
     let internalToken;
     try {
       internalToken = await exchangeForInternalToken(oauthCredentials);
     } catch (error) {
       ctx.log.error({ kibana: `OAuth Exchange Failed: ${error.message}` });
       ctx.status = error.response ? error.response.status : 500;
       ctx.body = error.response
         ? error.response.data
         : { error: "OAuth Exchange Failed" };
     }
     ctx.status = 200;  // ❌ Sobrescribe el status de error
     ctx.body = internalToken;  // ❌ undefined si hubo error
     ```

   - **Solución sugerida**: Agregar `return` después de setear la respuesta de error para evitar que el código continúe.

     ```javascript
     let internalToken;
     try {
       internalToken = await exchangeForInternalToken(oauthCredentials);
     } catch (error) {
       ctx.log.error({ kibana: `OAuth Exchange Failed: ${error.message}` });
       ctx.status = error.response ? error.response.status : 500;
       ctx.body = error.response
         ? error.response.data
         : { error: "OAuth Exchange Failed" };
       return;  // ✅ Detiene la ejecución aquí
     }
     ctx.status = 200;
     ctx.body = internalToken;
     ```


   - **Solución sugerida**: Agregar logging usando `pushLog` para trazabilidad.

     ```javascript
         "unknown",
         "unknown"
       );
       return {
         encrypted: "",
         iv: "",
         authTag: "",
         key: ""
       };
     }
     ```

## 📋 Resumen del PR

**¿Qué hace este PR?**
Este PR migra la configuración de la aplicación desde archivos `env.yaml` estáticos hacia variables de entorno (`APP_CONFIG`). Elimina credenciales hardcodeadas del repositorio, agrega un nuevo endpoint `/token-exchange` para intercambio de tokens Azure AD, y añade funcionalidad de encriptación de secrets para SalesForce y Android.

**Archivos principales modificados:**
- [DELETE] `api-auth/env.yaml` - Eliminación de credenciales hardcodeadas
- [DELETE] `providers/adfs-auth/env.yaml` - Eliminación de credenciales
- [DELETE] `providers/basic-auth/env.yaml` - Eliminación de credenciales
- [NEW] `api-auth/src/utils/env.js` - Nueva utilidad de configuración
- [NEW] `api-auth/src/utils/azureAuth.js` - Validación con Microsoft Graph
- [NEW] `api-auth/src/utils/encrypt-secrets.js` - Encriptación de secrets
- [NEW] `api-auth/src/helpers/encryptionHelper.js` - Helper de encriptación AES-256-GCM
- [MODIFY] `api-auth/src/controllers/Auth.js` - Nuevo endpoint tokenExchange + secrets en response

**¿Qué Podría impactar?**
- **Flujo de autenticación**: El bug P0 en tokenExchange causaría que errores de OAuth se reporten como éxito.
- **Despliegue**: Requiere que la variable de entorno `APP_CONFIG` esté configurada en todos los ambientes.
- **Integraciones**: El nuevo endpoint `/token-exchange` afecta flujos de autenticación con Azure AD.

**Impacto:**
- SEGURIDAD: Mejora significativa al eliminar credenciales del código fuente
- ARQUITECTURA: Cambio en la forma de obtener configuración en toda la aplicación
- FUNCIONALIDAD: Bug crítico en manejo de errores del endpoint tokenExchange
