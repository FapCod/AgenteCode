## 📋 Resumen del PR

**¿Qué hace este PR?**
Este PR migra la configuración de la aplicación desde archivos `env.yaml` estáticos hacia variables de entorno (`APP_CONFIG`). Esto elimina credenciales hardcodeadas del repositorio, mejorando significativamente la seguridad. Además, agrega un nuevo endpoint `/token-exchange` para intercambio de tokens Azure AD y funcionalidad de encriptación de secrets.

**Archivos principales modificados:**
- [DELETE] `api-auth/env.yaml` - Eliminación de credenciales hardcodeadas
- [DELETE] `providers/adfs-auth/env.yaml` - Eliminación de credenciales
- [DELETE] `providers/basic-auth/env.yaml` - Eliminación de credenciales
- [NEW] `api-auth/src/utils/env.js` - Nueva utilidad de configuración
- [NEW] `api-auth/src/utils/azureAuth.js` - Validación con Microsoft Graph
- [NEW] `api-auth/src/utils/encrypt-secrets.js` - Encriptación de secrets
- [NEW] `api-auth/src/helpers/encryptionHelper.js` - Helper de encriptación AES-256-GCM
- [MODIFY] `api-auth/src/controllers/Auth.js` - Nuevo endpoint tokenExchange
- [MODIFY] `api-auth/src/routes/auth.js` - Nueva ruta /token-exchange
- [MODIFY] Múltiples archivos - Migración de `yenv()` a `getEnv()`

**¿Qué Podría impactar?**
- **Despliegue**: Requiere que la variable de entorno `APP_CONFIG` esté configurada correctamente en todos los ambientes (QAS, PPR, PRD).
- **Dependencias**: Si `APP_CONFIG` no está presente, el sistema hace fallback a yenv (desarrollo local).
- **Integraciones**: El nuevo endpoint `/token-exchange` afecta flujos de autenticación con Azure AD.

**Impacto:**
- SEGURIDAD: Mejora significativa al eliminar credenciales del código fuente
- ARQUITECTURA: Cambio en la forma de obtener configuración en toda la aplicación
