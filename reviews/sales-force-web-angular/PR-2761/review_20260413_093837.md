## 🟡 Problemas Importantes (P1 - High Priority)

1. **Validacion insuficiente de credenciales OAuth**
   - **Archivo**: `ClientApp/projects/core/src/lib/rest/oauth.service.ts` linea ~12-16
   - **Problema**: El codigo emite un warning si las credenciales no estan presentes, pero continua la ejecucion creando un token Base64 con credenciales vacias (`btoa(\`${user}:${pwd}\`)`). Esto puede causar requests con autenticacion invalida que fallaran en el servidor.
   - **Impacto**: Requests silenciosamente fallidos, dificiles de debuggear en produccion.
   - **Impacto en score**: 4 puntos (Robustez)
   - **Tiempo est.**: 10 minutos
   - **Categoria afectada**: Robustez
   - **Codigo actual**:

     ```typescript
     const { user, pwd } = this.oauth;
     if (!user || !pwd) {
       console.warn('OAuth credentials not found in secrets');
     }
     const b64String = btoa(`${user}:${pwd}`);
     ```

   - **Solucion sugerida**: Lanzar un error o retornar un Promise rechazado si las credenciales no estan disponibles.

     ```typescript
     const { user, pwd } = this.oauth;
     if (!user || !pwd) {
       return Promise.reject(new Error('OAuth credentials not configured'));
     }
     const b64String = btoa(`${user}:${pwd}`);
     ```

2. **Error silenciado en saveSecrets (session.service.ts)**
   - **Archivo**: `ClientApp/projects/core/src/lib/services/session.service.ts`
   - **Problema**: Cuando falla el guardado de secretos, el error solo se loguea pero el flujo continua como si todo estuviera bien. Esto puede causar que los secretos no se persistan correctamente sin que el sistema lo detecte.
   - **Impacto**: Perdida silenciosa de secretos, sesion corrupta.
   - **Impacto en score**: 4 puntos (Robustez)
   - **Tiempo est.**: 15 minutos
   - **Categoria afectada**: Robustez
   - **Codigo actual**:

     ```typescript
     } catch (e) {
       console.error('Error saving secrets', e);
     }
     ```

   - **Solucion sugerida**: Propagar el error o retornar un indicador de fallo para que el llamador pueda manejarlo.

     ```typescript
     } catch (e) {
       console.error('Error saving secrets', e);
       throw e; // O retornar false si es metodo con return type
     }
     ```

## 📋 Resumen del PR
**Que hace este PR?**
Migra las credenciales sensibles (passwords, API keys, tokens de OAuth, claves de encriptacion) desde los archivos de environment hardcodeados hacia un sistema de secretos dinamico mediante `SecretService`. Los secretos ahora se obtienen de `SessionService` y se descifran usando Web Crypto API.

**Archivos principales modificados:**
- [NEW] `secrets.interface.ts` - Interface que define la estructura de los secretos
- [NEW] `secret.service.ts` - Servicio para acceder a los secretos de forma centralizada
- [MODIFY] `oauth.service.ts` - Ahora obtiene credenciales de SecretService
- [MODIFY] `auth-api-sb2.service.ts` - Ahora obtiene password de SecretService
- [MODIFY] `crypto.service.ts` - Nueva funcion decryptSecret con Web Crypto API
- [MODIFY] `session.service.ts` - Metodos para guardar/obtener secretos
- [MODIFY] `environment.*.ts` - Remocion de credenciales hardcodeadas
- [MODIFY] `.npmrc` - Uso de variables de entorno para registry y token

**Que Podria impactar?**
- Login y autenticacion OAuth (si los secretos no se cargan correctamente)
- Integracion con API SB2
- Funcionamiento del servicio de criptografia

**Impacto:**
- SEGURIDAD (mejora significativa al remover credenciales del codigo)
- ARQUITECTURA (nuevo patron de manejo de secretos)
