## 📋 Resumen del PR

**¿Que hace este PR?**
Migra las credenciales sensibles (passwords, API keys, tokens de OAuth, claves de encriptacion) desde los archivos de environment hardcodeados hacia un sistema de secretos dinamico mediante `SecretService`. Los secretos se obtienen de `SessionService` y se descifran usando Web Crypto API. Ademas, corrige los issues de validacion reportados en el review anterior.

**Archivos principales modificados:**
- [NEW] `secrets.interface.ts` - Interface que define la estructura de los secretos
- [NEW] `secret.service.ts` - Servicio para acceder a los secretos de forma centralizada
- [MODIFY] `oauth.service.ts` - Ahora valida credenciales y rechaza promesa si no estan disponibles
- [MODIFY] `auth-api-sb2.service.ts` - Ahora obtiene password de SecretService
- [MODIFY] `crypto.service.ts` - Nueva funcion decryptSecret con Web Crypto API
- [MODIFY] `session.service.ts` - Metodos para guardar/obtener secretos con propagacion de errores
- [MODIFY] `environment.*.ts` - Remocion de credenciales hardcodeadas
- [MODIFY] `.npmrc` - Uso de variables de entorno para registry y token
- [IGNORE] `.github/workflows/wf-security-branch.yml` - Archivo de CI/CD (no revisado)

**¿Que Podria impactar?**
- Login y autenticacion OAuth (depende de que los secretos se carguen correctamente)
- Integracion con API SB2
- Funcionamiento del servicio de criptografia

**Impacto:**
- SEGURIDAD (mejora significativa al remover credenciales del codigo)
- ARQUITECTURA (nuevo patron de manejo de secretos)
- ROBUSTEZ (mejor manejo de errores y validaciones)
