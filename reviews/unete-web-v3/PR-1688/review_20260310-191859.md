## �� Problemas Críticos (P0 - Blocking)

> *Issues de seguridad, crashes, o violaciones de reglas estrictas. Deben corregirse obligatoriamente.*

1. **[Posible Crash - Null Safety]**
   - **Archivo**: `packages/site/src/shared/utils/functions.js` línea 326 y `packages/site/src/screens/personal-address/pe/with-googlemap/step-information.jsx` línea 232
   - **Problema**: La función `cleanAddress` utiliza optional chaining (`rawAddress?.replace(...)`). Si recibe un valor nulo o `undefined` (como cuando no hay calle o componente de dirección), la función retorna `undefined`. En `step-information.jsx`, se llama a `cleanedAddress.indexOf(...)`. Al invocar este método sobre un `undefined`, se lanzará una excepción que romperá la aplicación en runtime (`TypeError: Cannot read properties of undefined (reading 'indexOf')`). Adicionalmente se rompe la regla de "Programación Defensiva - Parámetros sin validar".
   - **Riesgo**: La aplicación puede crashear impidiendo que el usuario complete su registro. (Bloqueo P0 de Null Safety).
   - **Impacto en score**: -16 puntos
   - **Tiempo est.**: 5 minutos
   - **Categoria afectada**: Robustez
   - **Código actual**:

     ```javascript
     function cleanAddress(rawAddress) {
       return rawAddress
         ?.replace(/\b[A-Z0-9]{4,8}\+[A-Z0-9]{2,3}\b[,\s]*/gi, '')
         .replace(/\s{2,}/g, ' ')
         .trim();
     }
     ```

   - **Solución sugerida**: Aplicar validación de parámetros al inicio del método para garantizar siempre el retorno de un tipo string.

     ```javascript
     function cleanAddress(rawAddress) {
       if (typeof rawAddress !== 'string' || !rawAddress) return '';
       return rawAddress
         .replace(/\b[A-Z0-9]{4,8}\+[A-Z0-9]{2,3}\b[,\s]*/gi, '')
         .replace(/\s{2,}/g, ' ')
         .trim();
     }
     ```

## 📋 Resumen del PR
**¿Qué hace este PR?**
Implementa una nueva utilidad `cleanAddress` que remueve los códigos "Plus Codes" (ej. `8Q7X+4W`) detectados por Google Maps y acorta espacios dobles, aplicando esta limpieza a las direcciones provenientes de geolocalización o inputs a lo largo de las vistas de distintos países (CO, CR, DO, GT, MX, PA, PE, PR, SV).

**Archivos principales modificados:**
- [MODIFY] `packages/site/src/shared/utils/functions.js` - Agregada utilidad `cleanAddress`.
- [MODIFY] `packages/site/src/screens/personal-address/*/step-information.jsx` - Reemplazo de variables locales por su versión sanitizada con `cleanAddress()`.

**¿Qué Podria impactar?**
El guardado de direcciones en el backend podría verse impactado enviando nulls corrompidos (`undefined` como string) si la dependencia base no es validada contra `null`/`undefined`. Así mismo, puede romper el flujo de UI en Perú si Google no devuelve una dirección correcta por fallo del parser.
