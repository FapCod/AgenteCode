## 📋 Resumen del PR
**¿Qué hace este PR?**
Corrige y mejora el manejo de errores en la generación de URLs firmadas de S3. Ahora se valida la extensión `png` por defecto y se retorna un mensaje detallado cuando una extensión es inválida o el flag de invalidez es disparado.

**Archivos principales modificados:**
- [MODIFY] `src/core/constants.js` - Se agregó `png` a las reglas de validación por defecto.
- [MODIFY] `src/core/s3.js` - Mejorado el mensaje de error de extensión e implementación de retorno de objeto de error detallado.
- [MODIFY] `src/modules/search/service.js` - Adaptación de la lógica para manejar el nuevo formato de retorno de `generateSignedUrl`.

**¿Qué Podria impactar?**
El cambio es seguro ya que se adaptaron los llamadores en la capa de servicios. Podría impactar positivamente en la experiencia del usuario al recibir errores más claros sobre qué extensiones se permiten.

**Impacto:**
- PERFORMANCE: Bajo
- ARQUITECTURA: Mejora en la comunicación de errores entre capas.
