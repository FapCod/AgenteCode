## 🔴 Problemas Críticos (P0 - Blocking)

1. **[Bug Lógico en Fallback de Feature Flag]**
   - **Archivo**: `app/Portal.Consultoras.WS.BusinessLogic/V1.1/OfertasBL.cs` (línea ~465)
   - **Problema**: En el método `SetComponentes`, si el país **es** Chile pero el **`featureFlagPPUM` está apagado** (`false`), el código no cumple ni entra a evaluar ninguna de las condiciones ni cuenta con un bloque `else` definitivo. Esto generará que silenciosamente no se asigne el string al `componente.Pum`. Un Feature Flag debe mantener siempre el comportamiento original y retrocompatible cuando se encuentra inactivo. 
   - **Riesgo**: Pérdida funcional de datos en la UI (desaparece el PUM) en caso de que el flag en Chile decida apagarse preventivamente tras el release e ignorar código.
   - **Impacto en score**: -15 puntos
   - **Código problemático**:
     ```csharp
     if (request.PaisISO.Equals(Constantes.PaisIso.Chile) && featureFlagPPUM) {
         // Logica Nueva...
     }
     else if (!request.PaisISO.Equals(Constantes.PaisIso.Chile)) {
         // Logica Original para resto de paises...
     }
     // ¿Y si ES chile pero el flag esta falso? 💥 No hace nada.
     ```
   - **Solución sugerida**: Reemplazar el `else if` con un `else` llano que provea el comportamiento de fallback sin exclusiones para preservar la compatibilidad en ON/OFF:
     ```csharp
     if (request.PaisISO.Equals(Constantes.PaisIso.Chile) && featureFlagPPUM)
     {
         componente.Pum = string.Format(Constantes.FichaProducto.FormatoPumCL, request.Simbolo, Util.DecimalToStringFormatByPais(precioContenidoTruncate, request.PaisISO), factorNormativo, componente.UnidadMedidaContenido);
         calculoPum = true;
     }
     else 
     {
         // Mantiene comportamiento original para todos, incluyendo CL con el Feature Flag apagado
         componente.Pum = string.Format(Constantes.FichaProducto.FormatoPum, componente.UnidadMedidaContenido, request.Simbolo, Util.DecimalToStringFormatByPais(precioContenidoTruncate, request.PaisISO));
         calculoPum = true;
     }
     ```

## 🟡 Problemas Importantes (P1 - High Priority)

1. **[Impacto Global no Anticipado en Helpers/Utilitarios]**
   - **Archivo**: `app/Portal.Consultoras.WS.Common/Util.cs`
   - **Problema**: Se ha modificado el formateo de la función estática generalizada `DecimalToStringFormatByPais` inyectando duro un retorno de formato de base `"n0"` para Chile, descartando los decimales en favor de enteros estáticos globalmente.
   - **Impacto**: Esta función es un Utilitario compartido transversalmente mediante Core Libraries, llamada en cascada a través de todo el Backend. Este cambio podría causar severas regresiones visuales o lógicas en el Portal para Chile en otros apartados no orientados al PPUM que en efecto debían poseer un visualizador `"n2"`.
   - **Solución recomendada**: Validar con QA u owner de negocio si el abandono de decimales en Chile se aprobó para un impacto Global. De tratarse únicamente de una necesidad de UX para el Feature actual, evitar alterar el Core Helper `Util.cs` y delegar el formateo `"n0"` específicamente a `OfertasBL` y `PedidoBL`.
