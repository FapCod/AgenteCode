   - **Tiempo est.**: 5 minutos
   - **Categoria afectada**: Robustez

## 🟡 Problemas Importantes (P1 - High Priority)
> *Problemas de performance, bugs probables o deuda técnica alta.*

1. **[Arquitectura] Instanciación manual de dependencia (Breaking DI)**
   - **Archivo**: `app/Portal.Consultoras.WS.BusinessLogic/V2.0/BannerBL.cs` línea 21
   - **Problema**: Se está instanciando `TablaLogicaBL` directamente en el constructor usando `new`. Esto rompe el patrón de Inyección de Dependencias y dificulta la realización de pruebas unitarias (Testability), ya que no se puede "mockear" la lógica de tablas lógicas.
   - **Impacto**: Deuda técnica y dificultad de mantenimiento.
   - **Impacto en score**: 10 puntos
   - **Tiempo est.**: 10 minutos
   - **Categoria afectada**: Mantenibilidad / Testabilidad
   - **Código actual**:

     ```csharp
     public BannerBL()
     {
         _tablaLogicaBL = new TablaLogicaBL();
     }
     ```

   - **Solución sugerida**: Inyectar la interfaz `ITablaLogicaBL` (si existe) o la clase a través del constructor.

     ```csharp
     public BannerBL(ITablaLogicaBL tablaLogicaBL)
     {
         _tablaLogicaBL = tablaLogicaBL;
     }
     ```
