## 🟡 Problemas Importantes (P1 - High Priority)
> *Problemas de performance, bugs probables o deuda técnica alta.*

1. **[Falta de `await` en llamada async]**
   - **Archivo**: `src/modules/android-secrets/graphql/queries.ts` línea 10
   - **Problema**: El método `getEncryptedSecrets()` es async pero se retorna sin `await`.
   - **Impacto**: Aunque GraphQL puede manejar Promises, esto puede causar comportamiento inesperado en logging de errores y timing.
   - **Impacto en score**: -4 puntos
   - **Tiempo est.**: 2 minutos
   - **Categoria afectada**: Funcionalidad
   - **Código actual**:

     ```typescript
     async AndroidSecrets() {
         try {
             return service.getEncryptedSecrets()
         } catch (error) {
     ```

   - **Solución sugerida**: Agregar `await` para ser explícito

     ```typescript
     async AndroidSecrets() {
         try {
             return await service.getEncryptedSecrets()
         } catch (error) {
     ```

---

2. **[Parsing de JSON en cada llamada sin caché]**
   - **Archivo**: `src/utils/env.ts` líneas 4-18
   - **Problema**: La función `getEnv()` parsea `process.env.APP_CONFIG` en cada invocación.
   - **Impacto**: En aplicaciones de alto tráfico, el parsing repetido de JSON puede impactar performance.
   - **Impacto en score**: -4 puntos
   - **Tiempo est.**: 15 minutos
   - **Categoria afectada**: Escalabilidad
   - **Código actual**:

     ```typescript
     function getEnv(): AppConfig {
       if (process.env.APP_CONFIG) {
         const configValue = process.env.APP_CONFIG.trim()
         // Parsea JSON cada vez que se llama
         const parsed = JSON.parse(configValue)
         return parsed
       }
       const env = yenv()
       return env
     }
     ```

   - **Solución sugerida**: Implementar singleton/caché

     ```typescript
     let cachedConfig: AppConfig | null = null

     function getEnv(): AppConfig {
       if (cachedConfig) {
         return cachedConfig
       }
       
       if (process.env.APP_CONFIG) {
         const configValue = process.env.APP_CONFIG.trim()
         if (!configValue) {
           throw new Error('APP_CONFIG environment variable is empty')
         }
         try {
           cachedConfig = JSON.parse(configValue)
           return cachedConfig!
         } catch (error) {
           throw new Error('Invalid JSON in APP_CONFIG environment variable')
         }
       }

       cachedConfig = yenv()
       return cachedConfig
     }
     ```

## 📋 Resumen del PR
**¿Qué hace este PR?**
Migra la obtención de configuración desde `yenv` (archivo `env.yaml`) hacia una nueva función `getEnv()` que permite:
