## 📋 Resumen del PR
**¿Qué hace este PR?**
Migra la obtención de configuración desde `yenv` (archivo `env.yaml`) hacia una nueva función `getEnv()` con caché singleton que permite:
1. Leer configuración desde variable de entorno `APP_CONFIG` (JSON) con validación
2. Fallback a `yenv()` si la variable no existe
3. Caché de configuración para evitar re-parsing

También agrega un nuevo módulo GraphQL `android-secrets` para exponer secretos encriptados a la app Android usando AES-256-GCM.

**Archivos principales modificados:**
- [NEW] `src/utils/env.ts` - Nueva función con caché singleton
- [NEW] `src/utils/interfaces/app-config.ts` - Interfaz TypeScript completa (129 líneas)
- [NEW] `src/utils/encryption-helper.ts` - Helper de encriptación AES-256-GCM
- [NEW] `src/modules/android-secrets/` - Módulo GraphQL para secrets
- [MODIFY] ~25 archivos - Cambio de `yenv` a `getEnv()`
- [DELETE] Referencia a `env.yaml` en Dockerfile y gulpfile
- [MODIFY] CI/CD workflow - Desactivación de flags de debug

**¿Qué Podría impactar?**
- Configuración del servidor si `APP_CONFIG` tiene formato incorrecto (ahora con validación)
- El nuevo endpoint GraphQL `AndroidSecrets` expone secretos encriptados

**Impacto:**
- SEGURIDAD: Alto (manejo de secretos y encriptación AES-256-GCM)
- PERFORMANCE: Positivo (caché de configuración implementado)
- ARQUITECTURA: Medio (nuevo patrón de configuración centralizado)
