## Propósito del PR

Este PR tiene como objetivo **mover secretos expuestos en el código fuente hacia AWS Secrets Manager**. Se implementa una estrategia de orquestación para obtener configuraciones sensibles (API keys, credenciales S3, tokens) desde AWS en lugar de tenerlas hardcodeadas en `Web.config`.

**Cambios principales:**
1. Nueva carpeta `SecretFactory/` con clases para gestión de secretos
2. Actualización del SDK de AWS de v2.3.55.2 a v3.7.500
3. Reemplazo de `ConfigurationManager.AppSettings[key]` por `AppSettingsOrchestrator.GetAppSetting(key)`
4. Eliminación de secretos hardcodeados del `Web.config`

#### Issue #3: Inicializador Estático sin Manejo de Errores

**File:** `app/PortalFFVVWeb/SecretFactory/AppSettingsOrchestrator.cs`  
**Line:** 12  
**Severity:** 🔴 **BLOCKING**  
**Rule:** Los inicializadores estáticos deben ser resilientes

**Description:**  
El código usa un inicializador estático que puede lanzar excepciones:
```csharp
private static readonly AwsSecretsManagerStrategy _strategy = new 
    AwsSecretsManagerStrategy(APPSETTINGS_SECRET_NAME);
```

Si AWS Secrets Manager no está disponible o mal configurado, esto causa `TypeInitializationException` que **NO puede ser capturada** en código llamador, causando que toda la aplicación falle al iniciar.

**Acción requerida:**  
Implementar lazy initialization con try-catch:
```csharp
private static readonly Lazy<AwsSecretsManagerStrategy> _strategy = 
    new Lazy<AwsSecretsManagerStrategy>(() => {
        try {
            return new AwsSecretsManagerStrategy(APPSETTINGS_SECRET_NAME);
        } catch (Exception ex) {
            // Log error, return null or fallback strategy
            return null;
        }
    });

public static string GetAppSetting(string key)
{
    if (_strategy.Value != null)
        return _strategy.Value.GetConnectionString(key);
    
    // Fallback to ConfigurationManager
    return ConfigurationManager.AppSettings[key];
}
```

#### Issue #4: Falta de Fallback cuando AWS No Está Disponible

**File:** `app/PortalFFVVWeb/SecretFactory/AwsSecretsManagerStrategy.cs`  
**Line:** 43-51  
**Severity:** 🔴 **BLOCKING**  
**Rule:** Los servicios externos deben tener fallbacks

**Description:**  
En `InitializeAwsClient()`, cuando las credenciales están vacías, se crea un cliente sin configuración:
```csharp
else
{
    return new AmazonSecretsManagerClient();
}
```

Sin `RegionEndpoint` configurado, esto falla con `"No RegionEndpoint or ServiceURL configured"`.

**Acción requerida:**  
```csharp
private IAmazonSecretsManager InitializeAwsClient()
{
    string accessKey = ConfigurationManager.AppSettings["AWS_ACCESS_KEY_ID_CUSTOM"];
    string secretKey = ConfigurationManager.AppSettings["AWS_SECRET_ACCESS_KEY_CUSTOM"];
    string region = ConfigurationManager.AppSettings["AWS_REGION"] ?? Constantes.AWS.RegionDefault;

    var regionEndpoint = RegionEndpoint.GetBySystemName(region);

    if (!string.IsNullOrEmpty(accessKey) && !string.IsNullOrEmpty(secretKey))
    {
        return new AmazonSecretsManagerClient(accessKey, secretKey, regionEndpoint);
    }
    
    // Use instance profile credentials with region
    return new AmazonSecretsManagerClient(regionEndpoint);
}
```
