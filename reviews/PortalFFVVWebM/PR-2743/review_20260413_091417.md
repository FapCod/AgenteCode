## 🟡 Problemas Importantes (P1 - High Priority)
> *Problemas de performance, bugs probables o deuda técnica alta.*

1. **[Binarios NuGet Commiteados en el Repositorio]**
   - **Archivo**: `app/packages/AWSSDK.*.dll`, `app/packages/AWSSDK.*.nupkg` (28+ archivos)
   - **Problema**: Los paquetes NuGet (dlls y nupkgs) están commiteados en el repositorio en lugar de ser restaurados vía `nuget restore`.
   - **Impacto**: Aumenta el tamaño del repositorio innecesariamente (~50+ MB de binarios), dificulta actualizaciones y causa conflictos de merge.
   - **Impacto en score**: -5 puntos
   - **Tiempo est.**: 15 minutos
   - **Categoria afectada**: Mantenibilidad
   - **Solución sugerida**: 
     1. Agregar `**/packages/` al `.gitignore`
     2. Eliminar la carpeta `packages/` del tracking de Git
     3. Asegurar que el pipeline de CI/CD ejecute `nuget restore` antes de compilar

---

2. **[Validación de AWS_SECRET_ENVIRONMENT Faltante]**
   - **Archivo**: `SecretFactory/AwsSecretsManagerStrategy.cs` línea 420-426
   - **Problema**: Si `AWS_SECRET_ENVIRONMENT` no está configurado en el Web.config, el nombre del secreto será `ffvv/portalffvvwebm/appsettings/` (con placeholder vacío), lo cual fallará silenciosamente o buscará un secreto incorrecto.
   - **Riesgo**: En nuevos ambientes o deployments incompletos, la aplicación puede comportarse de manera impredecible.
   - **Impacto en score**: -4 puntos
   - **Tiempo est.**: 10 minutos
   - **Categoria afectada**: Robustez
   - **Código actual**:

     ```csharp
     string secretEnvironment = ConfigurationManager.AppSettings["AWS_SECRET_ENVIRONMENT"];
     secretName = string.Format(secretName, secretEnvironment);
     
     if (string.IsNullOrEmpty(secretName)) // Esta validación nunca será true si el template tiene texto
     ```

   - **Solución sugerida**: Validar `secretEnvironment` antes del format

     ```csharp
     string secretEnvironment = ConfigurationManager.AppSettings["AWS_SECRET_ENVIRONMENT"];
     if (string.IsNullOrEmpty(secretEnvironment))
     {
         throw new ArgumentNullException(nameof(secretEnvironment), 
             "La variable AWS_SECRET_ENVIRONMENT no está configurada en Web.config");
     }
     secretName = string.Format(secretName, secretEnvironment);
     ```

   - **Archivo**: `SecretFactory/AwsSecretsManagerStrategy.cs` línea 489
   - **Impacto en score**: -1 punto
   - **Tiempo est.**: 5 minutos
   - **Categoria afectada**: Mantenibilidad
   - **Problema**: El `throw new ArgumentNullException` con mensaje "Retornando null" es capturado inmediatamente por el catch, loggeado, y luego retorna null. El throw es innecesario y el mensaje es confuso.
   - **Código actual**:

     ```csharp
     throw new ArgumentNullException(nameof(connectionKeyName), 
         $"La clave '{connectionKeyName}' no se encontró en el secreto AWS '{_secretName}'. Retornando null.");
     ```

   - **Solución sugerida**: Loguear directamente sin throw

     ```csharp
     if (_connectionStringsCache != null && _connectionStringsCache.TryGetValue(connectionKeyName, out string value))
     {
         return value;
     }
     ErrorUtilities.AddLog(null, $"La clave '{connectionKeyName}' no se encontró en '{_secretName}'", string.Empty);
     return null;
     ```
