3. **[Inicializador Estático sin Manejo de Errores]**
   - **Archivo**: `SecretFactory/AppSettingsOrchestrator.cs` línea 12
   - **Problema**: El inicializador estático puede lanzar `TypeInitializationException` que NO puede capturarse.
   - **Riesgo**: Si AWS Secrets Manager falla, toda la aplicación crashea sin posibilidad de recuperación.
   - **Impacto en score**: -20 puntos
   - **Tiempo est.**: 30 minutos
   - **Categoria afectada**: Robustez
   - **Código actual**:

     ```csharp
     private static readonly AwsSecretsManagerStrategy _strategy = new 
         AwsSecretsManagerStrategy(APPSETTINGS_SECRET_NAME);
     ```

   - **Solución sugerida**: Usar `Lazy<T>` con try-catch y fallback a ConfigurationManager

     ```csharp
     private static readonly Lazy<AwsSecretsManagerStrategy> _strategy = 
         new Lazy<AwsSecretsManagerStrategy>(() => {
             try {
                 return new AwsSecretsManagerStrategy(APPSETTINGS_SECRET_NAME);
             } catch (Exception ex) {
                 Trace.TraceError($"AWS Secrets Manager no disponible: {ex.Message}");
                 return null;
             }
         });

     public static string GetAppSetting(string key)
     {
         if (_strategy.Value != null)
             return _strategy.Value.GetConnectionString(key);
         return ConfigurationManager.AppSettings[key];
     }
     ```

## 📋 Resumen del PR
**¿Qué hace este PR?**
Migra secretos y configuraciones sensibles (API keys de AWS S3, tokens JWT, credenciales de servicios) desde `Web.config` hacia AWS Secrets Manager, implementando clases orquestadoras para obtener valores de forma segura.

**Archivos principales modificados:**
- [NEW] `SecretFactory/AppSettingsOrchestrator.cs` - Orquestador para obtener AppSettings desde AWS
- [NEW] `SecretFactory/AwsSecretsManagerStrategy.cs` - Estrategia de conexión a AWS Secrets Manager
- [NEW] `SecretFactory/SecurityOrchestrator.cs` - Orquestador para claves de seguridad
- [MODIFY] `Web.config` - Eliminación de secretos hardcodeados
- [MODIFY] Controllers y Providers - Uso de nuevos orquestadores

**¿Qué Podría impactar?**
- Toda la aplicación si AWS Secrets Manager no está disponible o mal configurado
- Funcionalidades que dependan de los 17+ secretos eliminados del Web.config

**Impacto:**
