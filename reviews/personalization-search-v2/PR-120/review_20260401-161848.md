- **Tiempo est.**: 5 minutos
- **Codigo actual**:

### 2. Uso de `.map()` con side-effects en lugar de `.forEach()`
- **Archivo**: `app/services/productApi.js` lineas 249-266 del diff
- **Problema**: `.map()` se usa solo para mutar `componente`, descartando el array resultante. Semanticamente correcto es `.forEach()`.
- **Impacto**: Crea un array temporal en memoria que se descarta. Comunica mal la intencion del codigo.
- **Impacto en score**: -2 puntos (Mantenibilidad)
- **Tiempo est.**: 5 minutos
- **Codigo actual**:
``````js
estrategia.Componentes.map((componente) => { /* mutaciones... */ });
``````
- **Solucion sugerida**:
``````js
estrategia.Componentes.forEach((componente) => { /* mutaciones... */ });
``````

### 3. `console.log` en catch block — nivel de log incorrecto
- **Archivo**: `app/controllers/oferta.js` linea 75 del diff
- **Problema**: `console.log(e)` usa nivel incorrecto para errores. No se detecto logger centralizado en el proyecto.
- **Impacto**: En produccion, logs de nivel `log` pueden filtrarse o perderse.
- **Impacto en score**: -2 puntos (Robustez)
- **Tiempo est.**: 2 minutos
- **Codigo actual**:
``````js
} catch (e) {
    console.log(e);
    next(e);
}
``````
- **Solucion sugerida**: Usar `console.error` para errores, o el logger centralizado del proyecto si existe:
``````js
} catch (e) {
    console.error(e);
    next(e);
}
``````

## Aprendizajes Clave
- Evitar comentarios en codigo nuevo: si necesita comentario, el nombre del metodo no es suficientemente descriptivo.
- Preferir nombres autoexplicativos. `policy.calculate()` y `policy.normalizeUnidad()` ya describen la intencion.
- Evitar `.map()` cuando la intencion es iterar con side-effects. Usar `.forEach()` para comunicar claramente que no se usa el resultado.

## Resumen del PR
**Que hace este PR?**
Implementa el calculo del Precio por Unidad de Medida (PUM) para productos: nuevo endpoint GET `/Oferta/PumCalculation/{codigoPais}/{codigoCampania}/{cuv}/{codigoConsultora?}`, capa de politicas por pais con patron Strategy (`pumPolicies.js`), e integracion con PAPI. Soporta Chile (CL) con conversion de unidades (L a ml, KG a g) y paises por defecto con comportamiento estandar.

**Archivos principales modificados:**
- [NEW] `app/services/pumPolicies.js` - Patron Strategy para politicas PUM por pais
- [MODIFY] `app/controllers/oferta.js` - Nuevo handler GetPumCalculation
- [MODIFY] `app/routes/oferta.js` - Nuevo endpoint con spec Swagger completa
- [MODIFY] `app/services/ofertaCuv.js` - Nuevo metodo SearchByCuvForPum
- [MODIFY] `app/services/productApi.js` - Integracion de policies en FillPum y FillDataEstrategia

**Que podria impactar?**
La funcion `FillPum` modificada es el core del calculo PUM para TODOS los paises. Un error en `shouldFill` o `calculate` del `defaultPolicy` puede afectar el calculo de PUM en los paises ya en produccion, no solo Chile.

**Impacto:**
- PERFORMANCE: Se agrega un `.map()` extra sobre `estrategia.Componentes` en cada llamada a `FillDataEstrategia`
- ARQUITECTURA: Patron Strategy mejora la extensibilidad por pais
