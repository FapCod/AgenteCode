## 🟡 Problemas Importantes (P1 - High Priority)

1. 💡 **[`console.log` en catch block — nivel de log incorrecto (Persistencia de Issue)]**
   - **Archivo**: `app/controllers/oferta.js` línea 71
   - **Problema**: El catch del endpoint nuevo `GetPumCalculation` imprime los errores como info estándar `console.log(e)` en lugar de loggearlos como errores críticos. Se pueden perder en infraestructuras de logging tipo CloudWatch/Splunk donde se busca por severidad.
   - **Impacto en score**: -2 puntos
   - **Tiempo est.**: 2 minutos
   - **Categoría afectada**: Robustez
   - **Código actual**:
     ```javascript
     } catch (e) {
         console.log(e);
         next(e);
     }
     ```
   - **Solución sugerida**: Reemplazar la escritura estándar del proceso por uso correcto del stream stderr.
     ```javascript
     } catch (e) {
         console.error(e);
         next(e);
     }
     ```
