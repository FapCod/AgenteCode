## 📋 Resumen del PR

**¿Qué hace este PR?**
Corrige el error HTTP 405 que ocurría cuando los usuarios realizaban búsquedas de productos con caracteres especiales (+, %, &) en la URL. El fix implementa:
1. Encoding de parámetros de búsqueda en JavaScript con `encodeURIComponent()`
2. Validación en el controlador C# para evitar redirección a URLs amigables cuando contienen caracteres especiales

**Archivos principales modificados:**
- [MODIFY] `BusquedaProductosController.cs` - Agrega validación de caracteres especiales antes de redireccionar
- [MODIFY] `BuscadorModule.js` - Aplica encoding a los parámetros de búsqueda

**¿Qué podría impactar?**
- Búsquedas de productos que contengan +, %, & ahora funcionarán correctamente
- No se espera impacto negativo en búsquedas normales

**Impacto:**
- FUNCIONALIDAD: Restaura funcionalidad de búsqueda con caracteres especiales
- SEGURIDAD: El encoding previene potenciales problemas de interpretación de URLs
