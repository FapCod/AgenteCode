## 🟡 Problemas Importantes (P1 - High Priority)
> *Problemas de performance, bugs probables o deuda técnica alta.*

1. **Breaking Change en `store()` — JSON.stringify para valores primitivos**
   - **Archivo**: `packages/web/src/shared/utils/helpers.ts` líneas 1031-1032
   - **Problema**: El setter de `store()` fue cambiado de `typeof value === 'object' ? JSON.stringify(value) : String(value)` a `JSON.stringify(value)`. Ahora los valores primitivos (strings, números, booleans) se serializan como JSON: la cadena `'hello'` se guarda como `'"hello"'`. Todo código existente que llame `store({ key, value: 'string' })` puede romperse silenciosamente.
   - **Impacto**: Breaking change de alto blast radius en toda la aplicación web. Los lectores que asuman un string plano obtendrán el valor entre comillas.
   - **Impacto en score**: -5 puntos (Funcionalidad, Robustez)
   - **Tiempo est.**: 15 minutos
   - **Categoria afectada**: Funcionalidad, Robustez
   - **Código actual**:

     ```typescript
     // NUEVO (potencialmente rompe primitivos)
     const valueToStore = JSON.stringify(value);
     ```

   - **Solución sugerida**: Revertir el cambio o verificar todos los callers de `store()` que usen valores primitivos.

     ```typescript
     // ANTERIOR (correcto para todos los tipos)
     const valueToStore = typeof value === 'object' ? JSON.stringify(value) : String(value);
     ```

2. **Código Duplicado — `openChatbot` persiste en `DrawerMenu`**

   - **Impacto**: Dos fuentes de verdad para la misma lógica. Si se modifica el hook, `DrawerMenu` quedará desactualizado silenciosamente.
   - **Impacto en score**: -7 puntos (Mantenibilidad, Arquitectura)
   - **Tiempo est.**: 20 minutos
   - **Categoria afectada**: Mantenibilidad, Arquitectura
   - **Código actual**:

       // ...misma lógica que en useChatbot...

## 📋 Resumen del PR
**¿Qué hace este PR?**
Implementa soporte para el nuevo avatar/imagen ISA del asistente virtual chatbot. Agrega constantes de configuración en `ConsTablaLogica.cs`, nuevas propiedades (`messageImage`, `flagNewImage`) en los schemas C# y TypeScript, un componente de ícono SVG (`IsaMessageIcon`), y una burbuja de mensaje animada en el chatbot JS legacy. En React, extrae la lógica del chatbot al hook `useChatbot` y añade el nuevo avatar ISA en el menú hamburguesa y la vista de asistente virtual.

**Archivos principales modificados:**
- [MODIFY] `ConsTablaLogica.cs` - Nuevas constantes ImageIsa, MessageMobile, MessageDesktop, FlagImageIsa
- [MODIFY] `UserLogic.cs` - Lógica de messageIsa según flag de configuración
- [MODIFY] `ChatbotController.cs` - Soporte para flagNewImage en configuración KoreAI
- [MODIFY] `chatbot.js` - ISA_BUBBLE_CONFIG, animación de burbuja, analytics de click
- [NEW] `use-chatbot.ts` - Hook centralizado para lógica del chatbot
- [NEW] `isa-message-icon.tsx` - SVG del ícono de mensaje ISA
- [MODIFY] `drawer-menu/index.tsx` - Integración del chatbot ISA en menú hamburguesa
- [MODIFY] `virtual-assistant/index.tsx` - Uso del hook useChatbot y nuevo diseño con imagen ISA

**¿Qué podría impactar?**
El cambio en `store()` puede afectar cualquier funcionalidad que use esta utilidad para almacenar valores primitivos en sessionStorage. El encoding corrupto en los keys de analytics impactará la calidad de los datos en GA4/GTM.
