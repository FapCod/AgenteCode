## 🔴 Problemas Críticos (P0 - Blocking)

> *Issues de seguridad, crashes, o violaciones de reglas estrictas. Deben corregirse obligatoriamente.*

1. **[Variable local en PascalCase]**
   - **Archivo**: `packages/app/Portal.Consultoras.Web/Controllers/PedidoRegistroController.cs` línea 1672
   - **Problema**: La variable local `PrecioContenidoStr` está declarada en `PascalCase`. Según las reglas P0 de errores de código, las variables locales deben usar `camelCase`. Solo las clases, propiedades y métodos deben usar `PascalCase`.
   - **Riesgo**: Violación directa a los estándares en .NET de nombrado de parámetros/locales en C#, disminuyendo la mantenibilidad del código base.
   - **Impacto en score**: -10 puntos
   - **Tiempo est.**: 2 minutos
   - **Categoria afectada**: Robustez
   - **Código actual**:

     ```csharp
     var PrecioContenidoStr = formateo.ToString("n0", CultureInfo.CreateSpecificCulture(string.Format("es-{0}", userData.CodigoISO)));
     ```

   - **Solución sugerida**: Cambiar el nombre a `camelCase` (`precioContenidoStr`).

     ```csharp
     var precioContenidoStr = formateo.ToString("n0", CultureInfo.CreateSpecificCulture(string.Format("es-{0}", userData.CodigoISO)));
     ```

1. **[Syntax Error en estilos (Bug de Layout)]**
   - **Archivo**: `packages/web/src/shared/components/organisms/modals/transactional-modals/suggested-cart/style.ts` línea 656
   - **Problema**: Existe un separador `;` inválido justo antes de `!important;` (`line-height: 16px; !important;`).

   - **Impacto en score**: -8 puntos
   - **Tiempo est.**: 2 minutos
   - **Categoria afectada**: Funcionalidad
   - **Código actual**:

     ```ts

     ```

   - **Solución sugerida**: Eliminar el `;` antes del importante.

     ```ts
     line-height: 16px !important; /* 133.333% */
     ```

2. **[Cálculo redundante y doble asignación (Rendimiento)]**
   - **Archivo**: `packages/app/Portal.Consultoras.Web/Providers/EstrategiaComponenteProvider.cs` línea 290
   - **Problema**: Se invoca inútilmente a `.ToString("n2", ...)` para crear un string inicial que es instantáneamente sobrescrito dos líneas después por `.ToString(formatStr, ...)` sin haber ocupado el primero en nada.
   - **Impacto**: Se aloja un String temporal en memoria y usa tiempo de CPU que nunca es utilizado, agregando deuda técnica.
   - **Impacto en score**: -6 puntos
   - **Tiempo est.**: 3 minutos
   - **Categoria afectada**: Mantenibilidad
   - **Código actual**:

     ```csharp
     c.PrecioContenidoStr = formateo.ToString("n2", CultureInfo.CreateSpecificCulture(string.Format("es-{0}", userData.CodigoISO)));
     var formatStr = userData.FFPPUM ? "n0" : "n2";
     c.PrecioContenidoStr = formateo.ToString(formatStr, CultureInfo.CreateSpecificCulture(string.Format("es-{0}", userData.CodigoISO)));
     ```

   - **Solución sugerida**: Realizar el formato usando la condicional directamente para asignar la propiedad una única vez.

     ```csharp
     var formatStr = userData.FFPPUM ? "n0" : "n2";
     c.PrecioContenidoStr = formateo.ToString(formatStr, CultureInfo.CreateSpecificCulture(string.Format("es-{0}", userData.CodigoISO)));
     ```
