## 🟡 Problemas Importantes (P1 - High Priority)
> *Problemas de performance, bugs probables o deuda técnica alta.*

1. **[Falso Positivo de Inicialización] Objeto de modelo creado vacío**
   - **Archivo**: `packages/app/Portal.Consultoras.Web/Controllers/PedidoRegistroController.cs` línea 197
   - **Problema**: Se inicializa la variable `ppum` usanto `new PpumData()`, causando que si el bloque `if (userData.FFPPUM)` no se ejecuta, el controlador devuelva un objeto vacío con datos `null` a la vista o endpoint JS, en lugar de no enviarlo. Esto causa en los componentes de Front-End (particularmente React localizados en el archivo `index.tsx`) que detecten la existencia del objeto (como Truthy) y pasen a generar rendering incompleto de textos con la variable como `(S/ por )`
   - **Impacto**: Interfaz gráfica corrupta o incorrecta (Riesgo de falso positivo/UI en el carrito para usuarios sin el feature flag).
   - **Impacto en score**: -10 puntos
   - **Tiempo est.**: 2 minutos
   - **Categoria afectada**: Funcionalidad
   - **Código actual**:

     ```csharp
     var ppum = new PpumData();
     if (userData.FFPPUM)
     {
         ppum = await ObtenerPPUMSiAplicaAsync(codigoIso, productoCarritoSugerido.CUV);
     }
     ```

   - **Solución sugerida**: Iniciar la variable `ppum` en valor `null` explícitamente y que solamente se inicialice con info real.

     ```csharp
     PpumData ppum = null;
     if (userData.FFPPUM)
     {
         ppum = await ObtenerPPUMSiAplicaAsync(codigoIso, productoCarritoSugerido.CUV);
     }
     ```
