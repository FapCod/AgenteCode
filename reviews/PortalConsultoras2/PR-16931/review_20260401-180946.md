## 🟡 Problemas Importantes (P1 - High Priority)
> *Problemas de performance, bugs probables o deuda técnica alta.*

1. **[Riesgo de Renderizado de un '0' residual en Componente React]**
   - **Archivo**: `packages/web/src/shared/components/organisms/modals/transactional-modals/suggested-cart/index.tsx` línea 165
   - **Problema**: En JavaScript/React al evaluar condiciones renderizadas usando el operador `&&` con un posible número con valor en cero (`content.ppum.factorNormativo && content.ppum.factorNormativo > 0 && ...`), se romperá el JSX y React emitirá e imprimirá ese "0" literalmente en la UI si el factor evaluado es 0 (cortocircutocircuito lógico Falsy en React).
   - **Impacto**: Bug visual al final en el carrito de compras donde el factor eventualmente se rinda a un 0 visual expuesto por error de componente.
   - **Impacto en score**: -5 puntos
   - **Tiempo est.**: 5 minutos
   - **Categoria afectada**: Funcionalidad / Robustez
   - **Código actual**:

     ```tsx
     {content.ppum && content.ppum.factorNormativo && content.ppum.factorNormativo > 0 && (
       <Typography className="suggested-cart-ppum-text" variant="p" weight={400}>
     ```

   - **Solución sugerida**: Forzar a que la validación condicional se resuelva como un check binario y utilizar un coalesce statement seguro omitiendo el primer condicional numérico flotante.

     ```tsx
     {(content.ppum?.factorNormativo ?? 0) > 0 && (
       <Typography className="suggested-cart-ppum-text" variant="p" weight={400}>
     ```
