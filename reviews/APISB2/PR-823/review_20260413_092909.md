## 🟡 Problemas Importantes (P1 - High Priority)

1. **[Llamadas .ToString() sobre valores potencialmente null]**
   - **Archivo**: Multiples archivos en `ServicioExterno/` y `TokenValidationHandler.cs`
   - **Problema**: Se llama `.ToString()` directamente sobre el resultado de `AppSettingsOrchestrator.GetAppSetting()` sin validar null primero. Si AWS Secrets falla y el ConfigurationManager tampoco tiene la key, retornara null y `.ToString()` lanzara `NullReferenceException`.
   - **Impacto**: Crash de la aplicacion al intentar autenticarse contra servicios externos (Buro de credito, Equifax, JWT).
   - **Impacto en score**: -4 puntos
   - **Tiempo est.**: 15 minutos
   - **Categoria afectada**: Robustez
   - **Codigo actual**:

     ```csharp
     // APCService.cs
     var sUsuario = AppSettingsOrchestrator.GetAppSetting("PA_USER_BURO").ToString();
     var sClave = AppSettingsOrchestrator.GetAppSetting("PA_CLAVE_BURO").ToString();

     // CirculoCredito.cs
     string clientID = AppSettingsOrchestrator.GetAppSetting("WSBuroMX_ClientID").ToString();

     // TokenValidationHandler.cs
     var secretKey = AppSettingsOrchestrator.GetAppSetting("JWT_SECRET_KEY");
     var securityKey = new SymmetricSecurityKey(System.Text.Encoding.Default.GetBytes(secretKey));
     ```

   - **Solucion sugerida**: Usar null coalescing o validar antes de usar

     ```csharp
     // Opcion 1: Null coalescing con string vacio
     var sUsuario = AppSettingsOrchestrator.GetAppSetting("PA_USER_BURO") ?? string.Empty;

     // Opcion 2: Validar y lanzar excepcion descriptiva
     var secretKey = AppSettingsOrchestrator.GetAppSetting("JWT_SECRET_KEY");
     if (string.IsNullOrEmpty(secretKey))
     {
         throw new ConfigurationErrorsException("JWT_SECRET_KEY no esta configurado en AWS Secrets ni AppSettings.");
     }
     var securityKey = new SymmetricSecurityKey(System.Text.Encoding.Default.GetBytes(secretKey));
     ```

2. **[Static initializer sin Lazy en SecretsBootstrap - Persiste]**
   - **Archivo**: `app/Belcorp.Api.Common/SecretFactory/SecretsBootstrap.cs` linea 13
   - **Problema**: El campo estatico `_strategy` se inicializa directamente sin `Lazy<T>`. Si AWS falla durante la inicializacion de la clase, lanzara `TypeInitializationException`.
   - **Impacto**: Crash de la aplicacion con excepcion poco descriptiva.
   - **Impacto en score**: -2 puntos
   - **Tiempo est.**: 10 minutos
   - **Categoria afectada**: Robustez
   - **Codigo actual**:

     ```csharp
     public static class SecretsBootstrap
     {
         private static readonly AwsSecretsManagerStrategy _strategy = new AwsSecretsManagerStrategy(awsSecretsName.CONNECTION_SECRET_NAME);
     ```

   - **Solucion sugerida**: Usar Lazy\<T\> con try-catch (igual que AppSettingsOrchestrator)

     ```csharp
     public static class SecretsBootstrap
     {
         private static readonly Lazy<AwsSecretsManagerStrategy> _strategy = new Lazy<AwsSecretsManagerStrategy>(() =>
         {
             try
             {
                 return new AwsSecretsManagerStrategy(awsSecretsName.CONNECTION_SECRET_NAME);
             }
             catch (Exception ex)
             {
                 LogManager.SaveLog(ex, "Error inicializando AWS Secrets para ConnectionStrings");
                 return null;
             }
         });

         public static void LoadConnectionStringsFromSecrets(bool isWinServ = false)
         {
             if (_strategy.Value == null)
             {
                 LogManager.SaveInfo("AWS Secrets no disponible, usando ConnectionStrings del config.");
                 return;
             }
             // ... resto del codigo
         }
     }
     ```

3. **[WebConfig.cs - Algunas propiedades aun retornan null - Persiste parcialmente]**
   - **Archivo**: `app/Belcorp.Api.Common/WebConfig.cs` multiples propiedades
   - **Problema**: Mientras `MandrillKey` fue corregido con `?? string.Empty`, otras propiedades como `SmtpLogin`, `SmtpPassword`, `AWSAccessKeyReadOnly`, etc. aun pueden retornar null.
   - **Impacto**: Codigo existente que asume strings no-nulos podria crashear.
   - **Impacto en score**: -2 puntos
   - **Tiempo est.**: 10 minutos
   - **Categoria afectada**: Robustez
   - **Codigo actual**:

     ```csharp
     public static string SmtpLogin
     {

     ```

## 📋 Resumen del PR

**¿Que hace este PR?**
Migra la gestion de secretos desde hardcoded en archivos de configuracion hacia AWS Secrets Manager. Implementa un sistema de cache con TTL configurable para los secretos y reemplaza todas las referencias a `ConfigurationManager.AppSettings["KEY"]` por `AppSettingsOrchestrator.GetAppSetting("KEY")`.

**Archivos principales modificados:**
- [NEW] `app/Belcorp.Api.Common/SecretFactory/AppSettingsOrchestrator.cs` - Orquestador de settings con Lazy
- [NEW] `app/Belcorp.Api.Common/SecretFactory/AwsSecretsManagerStrategy.cs` - Cliente AWS con cache
- [NEW] `app/Belcorp.Api.Common/SecretFactory/IConnectionStringStrategy.cs` - Interface Strategy
- [NEW] `app/Belcorp.Api.Common/SecretFactory/SecretsBootstrap.cs` - Bootstrap de connection strings
- [MODIFY] `app/Belcorp.Api/Web.config` - Remocion de secretos, placeholders
- [MODIFY] `app/Belcorp.Api.WindowsService/App.config` - Remocion de secretos
- [MODIFY] `app/Belcorp.Api.Common/WebConfig.cs` - Migracion a AppSettingsOrchestrator
- [MODIFY] `app/Belcorp.Api.BusinessLogic/ServicioExterno/*.cs` - Migracion a AppSettingsOrchestrator
- [MODIFY] `app/Belcorp.Api/Seguridad/TokenValidationHandler.cs` - JWT key desde Secrets

**¿Que podria impactar?**
- Conexiones a bases de datos de todos los paises (15 connection strings)
- Servicios de buro de credito (Equifax, Compliance, CirculoCredito, APC, DataCredito)
- Autenticacion JWT
- Integracion con AWS S3 y DynamoDB
- Servicios de email (Mandrill, SMTP)

**Impacto:**
- **Seguridad**: POSITIVO CRITICO - Elimina exposicion de secretos
- **Arquitectura**: POSITIVO - Centraliza gestion de secretos
- **Performance**: NEUTRAL - Cache evita llamadas excesivas a AWS
