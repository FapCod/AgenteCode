## 🟡 Problemas Importantes (P1 - High Priority)

1. **[Llamadas .ToString() sobre valores potencialmente null - Persiste]**
   - **Archivo**: Multiples archivos en `ServicioExterno/`
   - **Problema**: Se llama `.ToString()` directamente sobre el resultado de `AppSettingsOrchestrator.GetAppSetting()` sin validar null primero. Si AWS Secrets falla y el ConfigurationManager tampoco tiene la key, retornara null y `.ToString()` lanzara `NullReferenceException`.
   - **Impacto**: Crash de la aplicacion al intentar autenticarse contra servicios externos.
   - **Impacto en score**: -4 puntos
   - **Tiempo est.**: 20 minutos
   - **Categoria afectada**: Robustez
   - **Codigo actual**:

     ```csharp
     // BuroDominicana.cs
     consulta.ClientCredentials.UserName.UserName = AppSettingsOrchestrator.GetAppSetting("USERBURODO").ToString();
     consulta.ClientCredentials.UserName.Password = AppSettingsOrchestrator.GetAppSetting("PASSBURODO").ToString();

     // CirculoCredito.cs
     string clientID = AppSettingsOrchestrator.GetAppSetting("WSBuroMX_ClientID").ToString();
     string clientSecret = AppSettingsOrchestrator.GetAppSetting("WSBuroMX_ClientSecret").ToString();

     // Compliance.cs
     _user = AppSettingsOrchestrator.GetAppSetting("ComplianceUser").ToString();
     _pass = AppSettingsOrchestrator.GetAppSetting("CompliancePass").ToString();

     // DataCredito.cs
     string client_id = AppSettingsOrchestrator.GetAppSetting("WSBuroCO_CLIENT_ID").ToString();
     string client_secret = AppSettingsOrchestrator.GetAppSetting("WSBuroCO_CLIENT_SECRET").ToString();

     // Equifax.cs
     private string eqxCanal = AppSettingsOrchestrator.GetAppSetting("EqxCanal").ToString();
     private string eqxClave = AppSettingsOrchestrator.GetAppSetting("EqxClave").ToString();
     ```

   - **Solucion sugerida**: Usar null coalescing o validar antes de usar

     ```csharp
     // Opcion 1: Null coalescing (recomendado para settings opcionales)
     var userName = AppSettingsOrchestrator.GetAppSetting("USERBURODO") ?? string.Empty;

     // Opcion 2: Validacion con excepcion descriptiva (recomendado para settings criticos)
     var clientID = AppSettingsOrchestrator.GetAppSetting("WSBuroMX_ClientID");
     if (string.IsNullOrEmpty(clientID))
     {
         throw new ConfigurationErrorsException("WSBuroMX_ClientID no esta configurado en AWS Secrets ni AppSettings.");
     }
     ```

2. **[JWT_SECRET_KEY sin validacion null - Persiste]**
   - **Archivo**: `app/Belcorp.Api/Seguridad/TokenValidationHandler.cs` lineas 68, 133
   - **Problema**: `secretKey` se pasa directamente a `System.Text.Encoding.Default.GetBytes(secretKey)` sin validar que no sea null. Esto causara `ArgumentNullException`.
   - **Impacto**: Falla de autenticacion JWT si la configuracion no esta disponible.
   - **Impacto en score**: -2 puntos
   - **Tiempo est.**: 5 minutos
   - **Categoria afectada**: Robustez
   - **Codigo actual**:

     ```csharp
     var secretKey = AppSettingsOrchestrator.GetAppSetting("JWT_SECRET_KEY");
     var securityKey = new SymmetricSecurityKey(System.Text.Encoding.Default.GetBytes(secretKey));
     ```

   - **Solucion sugerida**: Validar antes de usar

     ```csharp
     var secretKey = AppSettingsOrchestrator.GetAppSetting("JWT_SECRET_KEY");
     if (string.IsNullOrEmpty(secretKey))
     {
         throw new ConfigurationErrorsException("JWT_SECRET_KEY no esta configurado en AWS Secrets ni AppSettings.");
     }
     var securityKey = new SymmetricSecurityKey(System.Text.Encoding.Default.GetBytes(secretKey));
     ```

## 📋 Resumen del PR

**Que hace este PR?**
Migra la gestion de secretos desde hardcoded en archivos de configuracion hacia AWS Secrets Manager. Implementa un sistema de cache con TTL configurable para los secretos y reemplaza todas las referencias a `ConfigurationManager.AppSettings["KEY"]` por `AppSettingsOrchestrator.GetAppSetting("KEY")`.

**Archivos principales modificados:**
- [NEW] `app/Belcorp.Api.Common/SecretFactory/AppSettingsOrchestrator.cs` - Orquestador de settings con Lazy
- [NEW] `app/Belcorp.Api.Common/SecretFactory/AwsSecretsManagerStrategy.cs` - Cliente AWS con cache
- [NEW] `app/Belcorp.Api.Common/SecretFactory/IConnectionStringStrategy.cs` - Interface Strategy
- [NEW] `app/Belcorp.Api.Common/SecretFactory/SecretsBootstrap.cs` - Bootstrap de connection strings con Lazy
- [MODIFY] `app/Belcorp.Api/Web.config` - Remocion de secretos, placeholders
- [MODIFY] `app/Belcorp.Api.WindowsService/App.config` - Remocion de secretos
- [MODIFY] `app/Belcorp.Api.Common/WebConfig.cs` - Migracion a AppSettingsOrchestrator con null coalescing
- [MODIFY] `app/Belcorp.Api.BusinessLogic/ServicioExterno/*.cs` - Migracion a AppSettingsOrchestrator
- [MODIFY] `app/Belcorp.Api/Seguridad/TokenValidationHandler.cs` - JWT key desde Secrets

**Que podria impactar?**
- Conexiones a bases de datos de todos los paises (15 connection strings)
- Servicios de buro de credito (Equifax, Compliance, CirculoCredito, APC, DataCredito)
- Autenticacion JWT
- Integracion con AWS S3 y DynamoDB
- Servicios de email (Mandrill, SMTP)

**Impacto:**
- **Seguridad**: POSITIVO CRITICO - Elimina exposicion de secretos
- **Arquitectura**: POSITIVO - Centraliza gestion de secretos
- **Performance**: NEUTRAL - Cache evita llamadas excesivas a AWS
