   - **Solución sugerida**: Usar `useCallback` para las funciones que se usan como dependencias

     ```typescript
             await instance.initialize();
             setAzureInit(true);
         } catch (err) {
             changeLoading(false);
         }
     }, [instance, changeLoading]);

     useEffect(() => {
         instance && !azureInit && initAzure();
     }, [instance, azureInit, initAzure]);
     ```

## 📋 Resumen del PR

**¿Qué hace este PR?**
Completa la migración del sistema de autenticación de OAuth client credentials a Azure AD SSO con MSAL (Microsoft Authentication Library). El flujo ahora utiliza token exchange donde el token de Azure AD se intercambia por un token de la API interna.

**Archivos principales modificados:**
- [MODIFY] `admin/.env*` - Eliminación de credenciales y nueva configuración de endpoints auth
- [MODIFY] `admin/src/api/AuthTokenApi.ts` - Refactor para recibir azureToken como parámetro
- [MODIFY] `admin/src/api/reqResAPI.ts` - Nuevo cliente api_auth, eliminación de api_oauth con Basic Auth
- [MODIFY] `admin/src/components/Organisms/Home.tsx` - Flujo completo de autenticación con MSAL
- [MODIFY] `admin/src/components/Organisms/Home2.tsx` - Sincronización con cambios de autenticación

**¿Qué podría impactar?**

- Endpoints de API (cambio a /ppr en QA y PPR)
