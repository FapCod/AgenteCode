## 🟡 Problemas Importantes (P1 - High Priority)

1. **[Catch blocks vacíos sin manejo de errores - Home.tsx]**
   - **Archivo**: `admin/src/components/Organisms/Home.tsx` líneas ~35, ~43
   - **Problema**: Las funciones `redirectEntraID` e `initAzure` tienen bloques catch completamente vacíos que silencian cualquier error sin ningún manejo ni feedback.
   - **Impacto**: Si Azure falla en inicializarse o el redirect falla, el usuario quedará en un estado de carga infinito sin saber qué pasó. Los errores se pierden silenciosamente.
   - **Impacto en score**: -3 puntos
   - **Tiempo est.**: 5 minutos
   - **Categoria afectada**: Robustez
   - **Código actual**:

     ```typescript
     const redirectEntraID = async () => {
         try {
             await instance.loginRedirect( loginRequest );
         } catch (err) {
         }
     }

     const initAzure = async () => {
         try {
             await instance.initialize();
             setAzureInit( true );
         } catch ( err ) {
         }
     }
     ```

   - **Solución sugerida**: Agregar manejo de estado de error o al menos cambiar el estado de loading para que el usuario sepa que algo falló

     ```typescript
     const redirectEntraID = async () => {
         try {
             await instance.loginRedirect( loginRequest );
         } catch (err) {
             changeLoading(false);
             // Opcional: setError(true) para mostrar mensaje al usuario
         }
     }

     const initAzure = async () => {
         try {
             await instance.initialize();
             setAzureInit( true );
         } catch ( err ) {
             changeLoading(false);
             // Opcional: setInitError(true) para mostrar mensaje al usuario
         }
     }
     ```

     ```typescript
             changeLoading(false);
         }
     ```

     ```typescript
         instance && !azureInit && initAzure();
     }, [instance, azureInit, initAzure]); // Sin necesidad de disable
     ```
