   - **Tiempo est.**: 3 minutos
   - **Categoria afectada**: Robustez
   - **Código actual**:

     ```typescript
     const validateToken = async () => {
         changeLoading( true );
         try {
             const response = await instance.acquireTokenSilent({
                 ...loginRequest,
                 account: accounts[0]  // ⚠️ Sin validación
             });
     ```

   - **Solución sugerida**: Agregar validación defensiva

     ```typescript
     const validateToken = async () => {
         if (accounts.length === 0) {
             changeLoading(false);
             return;
         }
         changeLoading( true );
         try {
             const response = await instance.acquireTokenSilent({
                 ...loginRequest,
                 account: accounts[0]
             });
     ```

## 📋 Resumen del PR

**¿Qué hace este PR?**
Migra el sistema de autenticación de OAuth client credentials (Basic Auth con usuario/contraseña hardcodeados) a un flujo de Azure AD SSO con MSAL. El nuevo flujo obtiene un token de Azure AD y lo intercambia por un token de la API mediante el endpoint `auth/token-exchange`.

**Archivos principales modificados:**
- [MODIFY] `admin/.env` - Remoción de credenciales, nueva config de auth
- [MODIFY] `admin/.env.*` - Actualización de todos los ambientes
- [MODIFY] `admin/src/api/AuthTokenApi.ts` - Nuevo parámetro azureToken
- [MODIFY] `admin/src/api/reqResAPI.ts` - Nuevo cliente api_auth
- [MODIFY] `admin/src/components/Organisms/Home.tsx` - Implementación MSAL
- [MODIFY] `admin/src/components/Organisms/Home2.tsx` - Sync con Home.tsx

**¿Qué podría impactar?**
- Flujo de login completo de la aplicación
- Sesiones de usuario existentes (requerirán re-login)
- Integración con Azure AD

**Impacto:**
- **Seguridad**: POSITIVO CRÍTICO - Elimina credenciales hardcodeadas
